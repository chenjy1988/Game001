## 二、NPC 培养系统

### 2.0 一句话

花 **金币 / 特殊物资 / 时间（周）**，对中后期加入或结识的 **非战斗 NPC**（或极少数特殊雇佣兵）持续投资，解锁 **营地能力** 与 **专属剧情事件**。

### 2.1 设计目标

1. **后期资源_sink**——Berwick 风经济紧缺下，给玩家「养人」的长期目标，而不只买装备。
2. **内容钥匙**——部分能力 / 支线 / 真结局前置 **只能** 通过培养解锁，避免纯战斗数值碾压。
3. **不抢战斗培养风头**——不上场的 NPC 提供 **情报 / 生产 / 事件 / 大地图便利**；战斗数值仍以雇佣兵 JP、装备为主（`weapon-system.md` 原则）。

### 2.2 培养对象（谁可被培养）

| 类型 | 示例 | 与后勤六职关系 |
|---|---|---|
| **功能型剧情 NPC** | 边塞译员、吐蕃僧、粟特账房、逃卒教头 | 可与军师/方士等 **同名不同人** 或 **额外支线人物** |
| **深化型后勤** | 已招募的铁匠，再开「秘传锻造」线 | 在学徒→大师之上叠 **专属培养树**（可选） |
| **阵营联络人** | 藩镇使者、义军头目 | 解锁合同类型 / 商店 / 事件池 |

**不做**：把每名雇佣兵都做成培养树（与 JP 重复）；不让培养对象上场战斗。

### 2.3 玩家操作（营地 UI 设想）

```
选择 NPC → 查看培养线（2~4 阶）→ 支付资源 + 等待 N 周 → 解锁节点
```

每个 **培养节点** 包含：

| 产出 | 说明 |
|---|---|
| **能力解锁** | 营地被动或大地图能力（见 2.4） |
| **特色事件** | 写入 `camp_events.json` 的 `requires_cultivation: node_id` |
| **可选：好感 / 立场** | 影响结局分支输入（与 `camp-events-system.md` 好感独立，可叠加） |

### 2.4 能力示例（占位，Phase 4 详定）

| NPC  archetype | 培养解锁示例 |
|---|---|
| 译员 | 异族合同翻译 → 俘虏审讯成功率 + 剧情选项 |
| 铁匠（深化） | 传说锻造配方 / 修理折扣 |
| 僧 / 道士 | 营地伤病恢复加速 / 一次战前占卜（士气或天气） |
| 联络人 | 新合同池、黑市、派系声望 |

### 2.5 资源与经济

- 主要消耗：**金币** + **传家宝 / 稀有物资**（与 `economy-system.md` 稀缺资源一致）。
- 可与 **周数** 绑定：培养进行中占用 NPC「忙碌」，不可并行多线大项目（防一键点满）。
- **不可逆**：节点解锁后不回退；换 NPC 不返还（鼓励选人培养）。

### 2.6 与事件系统衔接

```
camp_events 触发条件 += cultivation_node_unlocked("translator_t2")
特色事件文案 += 该 NPC 培养阶段专属分支
```

- 随机事件池 **不替代** 培养节点事件；培养事件 **更低频、高重量**。
- 背景 / 性格仍可影响 **选项文案**，见 `camp-events-system.md`。

### 2.7 实施阶段

| Phase | 内容 |
|---|---|
| **4** | 2~3 名示范 NPC、各 3 阶培养树、1 条能力 + 1 条特色事件链 |
| **5** | 扩展至 8~12 名、与主线章节 / 结局挂钩 |

**数据（规划）**：`data/npc_cultivation.json`（NPC id、节点、成本、解锁 `ability_id` / `event_id`）。

## 三、据点建造系统（远期设想 · 可能不做）

### 3.0 状态

> **仅为设想**，尚未纳入 Phase 4 必做范围。若工期或范围收紧，**优先做 NPC 培养 + 合同驱动据点**，不做自由建造。

### 3.1 设想内容

花资源在 **大地图节点** 建设或升级 **据点**（哨站、坞堡、商栈等）：

| 设想收益 | 说明 |
|---|---|
| 区域商店 / 招募池 | 该格周边合同奖励加成 |
| 存粮 / 减工资压力 | 经济缓冲（需谨慎，避免毁 Berwick 紧缺感） |
| 防御 / 事件 | 被围攻事件、守住解锁剧情 |

### 3.2 不做或弱化的理由（待评审）

- 与 **合同 / 主线占领据点** 功能重叠，易做成「刷格子」。
- 美术与 UI 成本高（地图据点状态、建造菜单）。
- 核心赌注在 **战斗 + 雇佣兵**，战略层宜轻。

### 3.3 若要做：简化原则

1. **预设据点槽**（5~8 个），非自由沙盒建造。
2. **三档升级**（破败 / 修整 /  fortified），每档 1~2 个被动。
3. **与培养联动**：某 NPC 培养节点 = 某据点升级前置。
4. **与不做时的替代**：据点状态仅由 **主线 / 合同结果** 改写（剧情旗标），玩家不亲手点建造。

### 3.4 决策检查项（立项前）

- [ ] Phase 4 里程碑是否已有足够战斗 + 营地事件体量？
- [ ] 建造是否提供 **战斗外唯一** 价值（非培养 / 合同可替代）？
- [ ] 单局时长是否承受 +5~10h 管理负担？

**结论栏**（立项时填写）：`采纳 / 推迟 / 取消` — 当前：**推迟，以合同据点 + NPC 培养为主**。
