# 战斗系统调研文档集

本目录是为我们项目（暂代号 Game001）做战斗系统决策时的参考资料。
每份文档都按相同骨架来写，便于横向对比：

1. **游戏概述**——年代、子类、核心循环
2. **关键机制详解**——只挑"对设计有结构性影响"的机制
3. **为什么好玩**——抽象出真正的体验驱动力
4. **设计缺陷与争议**——批判性分析，不只看好话
5. **对我们项目的启发**——明确的"拿/弃/改"建议

## 横向对比表（机制级）

| 机制 | Battle Brothers | Berwick Saga | XCOM 2 | 我们当前 |
|---|---|---|---|---|
| 网格 | 六边形 | 六边形 | 方格 | 六边形 |
| 行动资源 | AP (按武器扣) + 疲劳 | 一动一移 (FE 系) | 2 动作点 | AP（按武器扣）+ 疲劳 |
| 伤害公式 | 三段式（甲→溢出→HP）| 减法（攻−防）| 减法 + 暴击/暴击 | 三段式 |
| 命中率 | 显式百分比 | 显式百分比 | 显式百分比 | 显式百分比 |
| RNG 修正 | 纯随机 | "手感优先"+反击翻盘 | 同步随机 + 难度倾斜 | 纯随机 |
| 部位伤害 | 头/身 | 头部暴击（减法制下意义有限）| 暴击系统 | 头/身 ✅ |
| 护甲 | 双血条独立消耗 | 直接进减法公式 | 单 HP，护甲减伤 | 双血条 ✅ |
| 反向影响 | 围攻 +5%/邻接 | 战马独立、地形细分 | 掩体 / 高低差 | 围攻 +5%/邻接 ✅ |
| 长期耗损 | 疲劳 + 永久伤口 + 装备耐久 | 受伤 + 武器耐久 + 战马 | 长休 / 疤痕 | 疲劳（待砍） |
| 时间压力 | 几乎无 | 援军节奏控制 | 倒计时回合 | 无 |
| 武器系统 | 关键字 + 数值 | 大量种类 + 关键字 | 武器档次 + 模组 | 数值光谱（待改关键字制）|

## 决策导图（落地到本项目）

我们的目标定位（用户原话）：
> "比战兄弟简单轻快一点，不要那么硬核和那么多参数，但是该有的战术深度和配合都是要有的。"

把三家长处映射到我们项目，得到 **"战兄弟骨架 + Berwick 的克制 + XCOM 的友善"** 三点取舍：

| 维度 | 学谁 | 怎么落地 |
|---|---|---|
| 战斗骨架 | Battle Brothers | 双血条 + 部位 + ZoC + 围攻加成（已实现）|
| 武器多样性 | Battle Brothers | 关键字制（粉碎/穿刺/撕裂/二段距）|
| 长期耗损 | Berwick Saga | 用永久伤口替代疲劳，匕首是主要触发器 |
| 战斗节奏 | XCOM 2 | 战斗目标 6-10 回合内结束（不做 30 回合围城）|
| 命中率体感 | XCOM (二代心理学) | 未来考虑伪随机保护：90% 不应该连 miss 两次 |
| 参数密度 | Berwick 反例 | 武器从 11 字段 → 3 数字 + 1 关键字 |
| RNG 哲学 | Berwick "手感优先" | 暴击/反击保留戏剧性，但避免一发翻盘 |
| 复杂度上限 | XCOM 2 警告 | 不做装备耐久、不做盾耐久独立资源 |

## 阅读建议

1. 第一次读，按 **battle-brothers → berwick-saga → xcom** 的顺序
   （从最复杂走到最简化，便于看到每一档"轻量化"的代价）。
2. 写到我们自己 `design.md` 的具体调整时，回查每份文档末尾的"对我们项目的启发"。
3. 这些文档**不是**用来神化任何一款游戏的，每篇都有"设计缺陷"一节——
   我们要做的是**拿优点、绕开缺点**，而不是模仿全貌。
