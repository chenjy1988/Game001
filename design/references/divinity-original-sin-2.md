# Divinity: Original Sin 2 — 战斗系统与整体设计调研

> Larian Studios 2017 年的 cRPG 神作；Swen Vincke 执导；BAFTA 最佳叙事。
> 是 **AP + 元素互动 + 双护甲**的工业级答案，**Larian 的代表作**（直到《博德 3》登顶）。
> Metacritic 92，玩家评分 9/10，**至今稳居 cRPG 史前三**。

## 一、游戏概述

| 项 | 内容 |
|---|---|
| 全名 | Divinity: Original Sin 2（DOS2，Definitive Edition）|
| 创作者 | Swen Vincke（执导，Larian 创始人）|
| 开发商 | Larian Studios（《博德 3》之父）|
| 发行 | 2017 PC / 2018 主机 / 2025 次世代更新 / 2026 计划任天堂 |
| 平台 | PC / PS / Xbox / Switch / iPad |
| 子类 | 等距视角 cRPG + 回合制战棋 |
| 战斗规模 | 4 人队伍 vs 4–8 |
| 单战时长 | 10–60 分钟 |
| 主线时长 | 单周目 70–100h；全分支 200h+ |
| 多人合作 | **完整 4 人合作 + PVP**——四个玩家可同时控制一名角色 |
| 宏观循环 | 探索（开放岛屿）→ 对话 → 战斗 → 等级/源点 → 下一区域 |

**设计目标**：用回合制战棋承载 **"你是神选者，争夺神性"** 的 cRPG ——
> "每个角色都是写好的人物（甚至骷髅、蜥蜴、亡灵），
> 选哪个 Origin 决定整个故事看起来不同。
> 战斗不是清场，是**用元素和地形让世界为你打仗**。"
> 玩法哲学：**AP + 双护甲 + 元素表面 = 让战场本身成为可塑武器**。

## 二、核心战斗循环

```
进入战斗 → 按主动性（Initiative）排队
    ↓
轮到自己 → 4 AP（默认）/ 6 AP（最高）
    ↓
花 AP：移动 / 攻击 / 技能 / 物品 / 元素铺设
    ↓
观察战场表面变化（火/水/电/油/血/毒）
    ↓
利用元素互动 → 击穿对应护甲 → 触发控制 → 击杀
    ↓
战利品（极其丰富）+ 经验
```

**关键设计哲学**：DOS2 是 **"战场即玩具箱"** ——
- 每场战斗的桌子可以拿来当武器
- 油桶可以踢去敌人脚下点燃
- 雨水可以铺成战场，再用电技能群体麻痹
- **环境互动比角色技能更重要**

## 三、关键机制详解

### 3.1 Action Points（AP）系统 —— 简化但深邃

**与战兄弟对照表**：

| 维度 | DOS2 | 战兄弟 / 我们 |
|---|---|---|
| 基础 AP | **4**（默认）| 9 |
| AP 上限 | 6（孤狼天赋）| 9 |
| 移动 AP | 1 AP / 米（按距离）| 2 AP / hex |
| 攻击 AP | 2 AP（普攻）| 3-6（武器） |
| 技能 AP | 1-3 AP（按强度）| 同上 |
| 跨回合存 AP | ❌ 用不掉浪费 | ❌ 同上 |

**关键设计点**：
- AP 总量小（4-6），**每个动作都是真实决策**——不会出现"AP 满地浪费"
- 数值精简，新手 5 分钟能上手
- 小数 AP 池让"差 1 AP 才能放大招" 成为常见戏剧时刻

#### AP 操控（核心爆发机制）

| 工具 | 效果 | 代价 |
|---|---|---|
| **孤狼（Lone Wolf）天赋**| 队伍 ≤ 2 人时，AP 上限 +2 + 属性 +5 | 限制队伍规模 |
| **肾上腺素（Adrenaline）**| 立即 +2 AP | 下回合 -2 AP |
| **处决者（Executioner）**| 击杀敌人后 +2 AP | 必须击杀 |
| **元素亲和（Elemental Affinity）**| 站在对应元素表面 → 法术 -1 AP | 需要先铺面 |
| **血肉献祭（Flesh Sacrifice）**| 即时 +1 AP | 失血 |
| **Apotheosis（神化）**| 临时不消耗源点 | 高 cooldown |

→ 这些**爆发组合**让 DOS2 玩家能策划"一回合连放 7 个技能"的精彩瞬间——但代价是**下次 AP 透支**。

### 3.2 双护甲系统 —— 最具识别度的设计

**所有伤害分为物理 / 魔法两类**，每类有独立护甲：

```
物理伤害 → 先扣 物理护甲 → 扣完后才扣 HP
魔法伤害 → 先扣 魔法护甲 → 扣完后才扣 HP
```

#### 控制效果与护甲挂钩（核心创新）

| 控制效果 | 触发条件 |
|---|---|
| 击倒 / 流血 / 致残 | **物理护甲必须先打空** |
| 眩晕 / 冰冻 / 燃烧 / 魅惑 | **魔法护甲必须先打空** |

→ 这一条规则**重塑了整个战斗节奏**：
- "无敌 BOSS 你打不动" → 必须先选定一种护甲集中打
- "硬控秒杀流" → 物理流的杀手锏
- "元素连锁" → 魔法流的优势

#### 双护甲的争议（设计代价）

**社区分歧明显**：
| 立场 | 论点 |
|---|---|
| 批评（新玩家居多）| 系统**强迫单一伤害类型**——3 物 1 魔的混合队没法施加任何控制 |
| 辩护（老玩家居多）| 均衡 2:2 完全可行；3:1 用召唤物补足 |
| 共识 | **3:1 配置确实较弱；纯单流派最强** |

→ 这是 DOS2 **最大的平衡缺陷**——也是它最具标识性的特征。

### 3.3 元素表面互动（环境即武器）

战场本身是动态的——**所有地面都可以铺成元素表面**，互相反应：

#### 互动表（玩家必背）

| 操作 | 结果 |
|---|---|
| 火球 + 油 | 大火（持续灼烧）|
| 雨水 + 火焰 | **蒸汽**（遮挡视野，施加近视）|
| 释电 + 水/蒸汽 | **电气化表面**（站上去眩晕）|
| 血雨 + 冰冻 | **血冰**（滑倒）|
| 血液 + 地系 | **毒液** |
| 全域冷却 | 所有水域秒变冰面 |

#### 战术应用层级

| 层 | 玩法 |
|---|---|
| 入门 | 火球点燃油桶 → 群伤 |
| 进阶 | 雨 → 电 → 群体麻痹 |
| 高级 | 全域冷却 → 冰面 → 滑倒控场 |
| 大师 | 血雨 → 冰冻 → 血冰 + 毒液 → 多层叠加 |

→ **环境是 DOS2 真正的"第五个角色"**——一场战斗失败常因为"踩了自己人铺的火"。

### 3.4 Source 源点技能 —— 终结手段

- **Source Points（源点）**：稀有资源，从 Source Pool 补充
- 源点技能不仅有冷却，还受**源点容量**限制
- 容量从 1 → 3，靠完成"导师任务"提升
- 源点技能是真正的**翻盘按钮**

代表技能：
| 技能 | 学派 | 效果 |
|---|---|---|
| **Time Warp**（时间扭曲）| 元素 | 让目标立即再行动一次 |
| **Dome of Protection**（保护穹顶）| 防护 | 区域无敌 buff |
| **Meteor Shower**（流星雨）| 火焰 | 全屏 AOE |
| **Grasp of the Starved**（饿者之握）| 死灵 | 无血单位强力 |

### 3.5 状态链式释放 —— "1 + 1 = 致命"

控制状态有**升级机制**：

| 前置状态 | 触发 | 升级 |
|---|---|---|
| Chilled（寒冷）| 再次冰冻 | **Frozen 冰冻**（跳过整个回合）|
| Shocked（触电）| 再次空气 | **Stunned 眩晕** |
| Wet（湿润）| 冰冻 | Frozen |
| Wet | 雷电 | Shocked |
| Decaying（腐朽）| 治疗 | **治疗反转为伤害** |

→ 高难度"战术家 / 荣誉模式"下，**精确叠状态成为生存必修课**。

### 3.6 Origin 起源角色 —— 叙事核心

**6 个预设主角**——每个都有**完整独立的剧情线**：

| Origin | 种族 | 特征 |
|---|---|---|
| **Fane**（费恩）| 永生族（骷髅）| 戴面具的学者，与七神时代有关 |
| **Lohse**（洛思）| 人类 | 被恶魔附身的吟游诗人 |
| **The Red Prince**（猩红王子）| 蜥蜴人 | 流亡王族，傲慢 |
| **Sebille**（赛比尔）| 精灵 | 前奴隶，复仇者 |
| **Beast**（野兽）| 矮人 | 流亡海盗船长 |
| **Ifan ben-Mezd**（伊凡）| 人类 | 神圣骑士团雇佣兵 |

#### 起源带来什么

- **Tags（标签）**：每个 Origin 有 5-8 个标签（"贵族"/"奴隶"/"叛徒"），开启专属对白
- **个人任务线**：每个 Origin 有 ~10h 的私人故事
- **专属结局**：选谁做主角影响"谁能成神"
- **互动尴尬**：把另一个 Origin 当队友 → 看到他的视角的故事
- **多人合作**：4 人合作时每人选一个 Origin → **4 条故事线同时推进**

→ 这是 DOS2 重玩价值的真正源头——**6 周目 = 6 个全新故事**。

### 3.7 多周目与对话

- **完整全语音**：每个对话都有专业声优
- **种族特性影响对话**：精灵能"读"人骨头看记忆，蜥蜴人能挖坟，狗能闻味
- **动物语**：人皮戒指可与动物对话——是 DOS 系列招牌
- **对话有重量**：选错 → 同伴脱队、隐藏剧情错过、永久后果
- **多种结局**：6 个 Origin × 多种关键选择 = 数十种结局变体

### 3.8 多人合作 —— DOS 系列 DNA

- **4 人合作 / 2 人对战**——所有内容都为多人设计
- **角色独立**：每个玩家**完全独立操作**自己角色，可以**对队友撒谎、偷东西、PVP**
- **裁决机制**：意见不合时投骰子或 PVP 决定
- **Game Master 模式**：让一个玩家当 DM 自定义剧情（仿 D&D）

→ **DOS2 是 cRPG 中最完整的多人合作实现**，《博德 3》直接继承。

## 五、设计缺陷与争议

### 5.1 双护甲的平衡问题（最大争议）
- **强迫单一伤害类型** —— 3:1 配置约等于残废
- 召唤物虽然能补，但**召唤系职业本身需占技能槽**
- 玩家社区分裂明显：老玩家说"是 feature"，新玩家说"是 bug"

### 5.2 难度曲线陡峭
- **第二章 Fort Joy** 是经典劝退点 —— 等级 3 的玩家碰上等级 5 的怪
- **战术家模式**几乎要**重新理解整个系统**
- "荣誉模式"（一命通关）只有 0.1% 玩家完成

### 5.3 第三章 / 第四章节奏崩坏
- 媒体共识：**第二章 Fort Joy 是巅峰**
- 第三章（Reaper's Coast）开放感强，但**任务密度过高**
- 第四章（Nameless Isle）短得反常 —— 像被砍过
- 第五章（Arx）任务过载，**结局区域整合不完美**

### 5.4 等级 vs 装备的过度敏感
- **怪等级 +2 = 玩家几乎打不动**
- 装备等级一过期立即报废
- 玩家被迫"刷副本"或"读档跳关"

### 5.5 物理流 > 魔法流（数值偏置）
- 玩家社区共识：**物理流通用且强**
- 魔法流碰到高魔抗 BOSS 极困难
- 战术家模式下 4 物理输出是最优解之一

### 5.6 经济崩坏
- **偷东西完全没代价**——所有 NPC 商店都能偷光
- 攻略派玩家通过偷窃可在第二章拥有"无限金币"
- 削弱了金币管理本应有的张力

### 5.7 GM 模式没多少人用
- **Game Master Mode** 概念惊艳但工具复杂
- 实际使用人数 < 1%
- 工具维护被砍，最终成"半成品功能"

### 5.8 UI 与控制不友好
- **背包管理灾难**——4 人队 × 数百件物品
- 寻路 AI 时不时卡住（"沿路绕开桌子"会花 3 秒）
- 主机版操作折磨（PC 鼠标键盘最佳）

### 5.9 战斗动画偶尔不可跳过
- 部分召唤动画 10+ 秒
- 多人合作时一人在动画里其他人发呆
- Larian 在博德 3 修复了这个

## 六、对我们项目的启发

### 6.1 直接拿（哲学层）
- ✅ **AP 总量精简的智慧**——DOS2 用 4 AP 做出战兄弟 9 AP 的深度，证明**"AP 多少不重要，比例和价值重要"**
  - 我们的 9 AP 是合理的，但可以考虑某些动作的 AP cost 重新平衡（参考 DOS2 的 1AP 移动 1m）
- ✅ **元素表面互动**——这是我们当前完全没有的维度
  - **未来 Phase 3-4 候选**：油桶、火堆、积水等**地形物件**作为可破坏/可点燃的战场互动
- ✅ **战场叙事化**——每个区域有它的故事，战斗目标多样化（不只杀光）
- ✅ **对话标签影响战斗** —— 种族/背景影响开场谈判，避战或诱降

### 6.2 直接拿（机制层）
- ✅ **AP 操控天赋**（处决者 +2 AP / 元素亲和 -1 AP）—— 给玩家"打破上限"的兴奋感
  - 我们的 trait 系统未来加几个 AP 操控类
- 🟡 **双护甲分类**（物理 / 魔法）—— 我们已经有"头/身护甲分离"，再加"魔法抗性"层会变第三轴
  - **不推荐照搬** ——双护甲是 DOS2 的标志，但它本身就是争议点
- ✅ **状态链式叠加**（湿润 → 冰冻 → 击倒）—— 是 SRPG 控制深度的工业方案
  - 我们未来加状态时可以考虑此类"前置 + 触发"模式
- ✅ **Origin 起源角色**（剧情驱动）—— 雇佣兵的"招募时"可以做小型 Origin 系统：每个雇佣兵有 background tag，进酒馆触发独特对话

### 6.3 警示（避开它的坑）
- ❌ **双护甲的 3:1 残废问题**——若引入要从一开始就平衡，绝不能让一种流派绝对优势
- ❌ **怪等级 +2 = 打不动**——避免"等级敏感性过高"，我们没传统等级（只有装备 + 武器熟练度），天然规避
- ❌ **偷窃无代价**——经济系统要有真实摩擦
- ❌ **后期章节节奏崩坏**——FFT/DOS2/TO 通病，**主线第 4 章前必须验证完整体验**
- ❌ **召唤动画 10+ 秒不可跳过**——所有动画必须可中断/加速
- ❌ **GM 模式 / 多人合作复杂度**—— Phase 3-4 之前不考虑
- ❌ **背包灾难**—— 4 人队物品管理就崩了，我们队伍如果扩到 6+ 必须有装备分类与快捷过滤

### 6.4 真正值得借鉴的"DOS2 精神"
1. **战场即玩具箱**——把地形从"装饰"变成"机制"。即便我们短期内不做元素互动，**油桶/火堆/破坏物**这种交互可以作为 Phase 3 候选
2. **AP 是节奏的钢琴键**——4 AP 就能打出 7 步连招，证明 AP 设计的精髓不在数量
3. **角色"是谁"决定剧情**——Origin/Tag 系统让每个角色独特，**雇佣兵不只是数值容器**
4. **失败的剧情感**——DOS2 的死亡有 mood，玩家会记住"那场雨电连锁我没准备好"，这是体验高光
5. **多人合作是黏性放大器**——长期看 cRPG/SRPG 的留存核心是合作；但 Phase 3+ 才考虑
6. **写作不是装饰，是机制**——种族对话、动物语、Tag 触发对白、读骨头记忆——这些**改变玩法的写作**比 cutscene 重要 10 倍

## 七、技能系统深度拆解

### 7.1 技能的六种来源

| 来源 | 获取方式 | 占记忆槽 | 需要能力点 | 可否遗忘 | 典型例子 |
|---|---|---|---|---|---|
| **技能书** | 购买/偷窃/拾取 → 阅读 | 1 槽 | ✅ 需满足对应学派最低要求 | ✅ 可随时替换 | 火球术、战吼 |
| **技能书合成** | 两本不同学派技能书组合 | 1 槽 | ✅ 需同时满足两种学派要求 | ✅ | 呼吸气泡（大气+战争）|
| **源力技能书** | 完成任务获得 | **2 槽** | ✅ 需满足学派要求 | ✅ | 流星雨、时间扭曲 |
| **装备赋予** | 穿戴特定装备自动获得 | **0 槽** | ❌ 不需要 | 脱装备即忘 | Heart of Kvyn 胸甲 |
| **种族/Origin 天赋** | 角色自带 | **0 槽** | ❌ | ❌ 永久 | 精灵食尸、蜥蜴人喷火 |
| **技能衍生** | 某技能施放后临时获得 | 临时槽 | — | 自动消失 | 展翅→飞行 |

**设计洞察**：
- **记忆槽是硬约束**：不是"学了就能用"，而是"学了还要记住"——Memory 属性决定了你能同时装备多少技能
- **装备技能不占槽**：巧妙地让装备不仅是数值，还提供战术选择——换装 = 换技能
- **合成技强制跨流派**：想学组合技？必须同时投资两个学派——这是 DOS2 推动 Build 多元化的核心杠杆

### 7.2 记忆槽系统——技能数量的硬约束

```
基础记忆槽 ≈ 3（起始）
每点 Memory 属性 → +1 槽
Mnemonic 天赋 → +3 槽
Memory 属性上限 ≈ 40（含装备加成）→ 最多 ~43 槽
```

| 技能类型 | 占槽 | 全场大约有多少个 | 典型配置占比 |
|---|---|---|---|
| 普通技能 | 1 | ~200+ | 大多数 |
| 源力技能 | 2 | ~20 | 大招 |
| 装备/天赋技能 | 0 | ~30 | 免费获得 |

**记忆槽的设计哲学**：
- **不是"学多少用多少"**——你可以在城镇学 50 个技能，但战斗中只能记住 ~10-15 个
- **Memory 属性与输出属性的竞争**：每投 1 点 Memory = 少投 1 点 Str/Fin/Int = 少 5% 伤害 → **技能多样性和输出效率是零和博弈**
- **这是 DOS2 最被低估的设计**：它让"我应该带什么技能进战斗"成为真正的策略决策

### 7.3 十大学派——技能分类体系

| 学派 | 主属性 | 伤害类型 | 对抗护甲 | 每点加成 | 定位 |
|---|---|---|---|---|---|
| **Warfare 战争** | Str/Fin/Int(依武器) | 物理(依武器) | 物理护甲 | **+5% 全物理伤害** | 物理输出基础 |
| **Huntsman 猎手** | Fin | 物理(依武器) | 物理护甲 | +5% 高地伤害 | 远程+高地 |
| **Scoundrel 恶棍** | Fin | 物理/穿刺 | 物理护甲 | +5% 暴击倍率 | 背刺+暴击 |
| **Pyrokinetic 烈火** | Int | 火焰 | 魔法护甲 | +5% 火伤 | AOE+点燃 |
| **Hydrosophist 水占** | Int | 水/物理 | 魔法/物理 | +5% 治疗/水伤 | 治疗+控场 |
| **Aerotheurge 大气** | Int | 空气 | 魔法护甲 | +5% 气伤 | 眩晕+位移 |
| **Geomancer 地卜** | Int | 地/毒 | 魔法护甲 | +5% 地伤+物理甲恢复 | 护甲+毒 |
| **Necromancer 死灵** | Int | **物理** | 物理护甲 | **+10% 生命偷取**（不加伤害）| 物理法术 |
| **Summoning 召唤** | Int | 物理/全元素 | 依召唤物 | +10% 召唤物生命/伤害 | 召唤 |
| **Polymorph 变形** | Str | 物理/地 | 物理/魔法 | **+1 属性点**（每点） | 自由变形 |

**三个关键设计洞察**：

1. **Warfare 是物理系的"元学派"**——它的 +5% 加成适用于**所有物理伤害**（包括死灵法术和武器攻击），所以物理角色必须先点满 Warfare 再考虑武器专精。这是一个**单一投资高回报**的设计，保证了物理系的统一性。

2. **死灵学派造成物理伤害却受智力加成**——这是"学派与属性的错位设计"，让智力型角色也能打物理伤害，打破了"智力=魔法"的刻板印象。

3. **变形学派每点给 +1 属性点**——不强化技能本身，而是给自由属性点。这让它成为"万金油"投资，任何 Build 都能从中获益。

### 7.4 技能伤害缩放——两种完全不同的公式

#### 物理技能（Warfare / Huntsman / Scoundrel）

```
最终伤害 = 武器基础伤害
         × (1 + 属性加成%)        // Str 或 Fin 每点 +5%
         × (1 + Warfare 加成%)    // 每点 Warfare +5%，优先计算
         × (1 + 武器专精加成%)    // 单手/双手/远程 各+5%/点
         × 技能倍率              // 各技能固定系数
         × 暴击倍率              // 默认 ×1.5
         × 其他修正
```

**核心**：武器伤害是地基，技能倍率是楼层。武器不强，技能再好也没用。

#### 魔法技能（Pyro / Hydro / Aero / Geo）

```
最终伤害 = 技能基础伤害            // 由角色等级决定
         × (1 + 智力加成%)        // 每点 Int +5%
         × (1 + 学派加成%)        // 对应学派每点 +5%
         × 暴击倍率              // 默认 ×1.5
         × 其他修正
```

**核心**：技能自带基础伤害，不依赖武器。智力型法杖不提供伤害加成，只提供属性门槛。

**两种缩放的根本差异**：

| 维度 | 物理技能 | 魔法技能 |
|---|---|---|
| 伤害地基 | **武器面板** | **角色等级** |
| 换武器 | 伤害巨变 | 几乎无影响 |
| 升级 | 武器必须更新 | 自动增长 |
| 空手 | 伤害极低 | 正常施法 |
| Build 核心 | 找好武器 | 投智力+学派 |

### 7.5 技能冷却与使用限制

| 维度 | 规则 |
|---|---|
| 每回合施放 | 每个技能**每回合只能施放一次** |
| 冷却回合 | 技能图标右下角数字 = 冷却回合数，通常 2-5 回合 |
| 战斗唯一 | 极少数技能（时间扭曲、神化）**整场战斗只能用一次** |
| 无冷却 | 极少数技能（法杖射击）每回合可用 |
| 源点消耗 | 源力技能额外消耗 1-3 点 Source Point |
| 状态封锁 | 沉默 = 不能施法术；狂暴 = 不能选目标 |

**AP 操控让"冷却"变得可利用**：
- 肾上腺素（+2AP 本回合，-2AP 下回合）让你**这一回合多放一个技能**
- 元素亲和（站在对应元素表面 → 法术 -1AP）让**原本 2AP 的法术变 1AP**
- 处决者（击杀 +2AP）让**击杀后额外多放一个技能**

→ 技能冷却 × AP 操控 × 元素表面 = **一回合放 7 个技能**的理论上限

### 7.6 组合技——强制跨流派的独特设计

DOS2 有 ~20 个**只能通过合成两本不同学派技能书**获得的技能：

| 组合 | 学派 A | 学派 B | 结果 |
|---|---|---|---|
| 呼吸气泡 | 大气 | 战争 | 水下呼吸+免疫窒息 |
| 闪烁 | 大气 | 恶棍 | 瞬移+背刺 |
| 火焰陷阱 | 烈火 | 猎手 | 铺设火焰陷阱 |
| 大地之怒 | 地卜 | 死灵 | 范围物理+减速 |
| 腐败皮肤 | 水占 | 死灵 | 治疗→伤害反转 |
| 血雨 | 水占 | 召唤 | 全屏下雨+召唤血幻 |

**设计哲学**：
- **强制跨流派投资**——你必须同时满足两种学派的最低能力点要求
- **不可能从 NPC 购买**——只能自己合成或随机拾取，增加了探索价值
- **源力版组合技更强**——用源力技能书作为合成材料，产物也更逆天

### 7.7 技能层级——解锁门槛

| 层级 | 所需学派点数 | 技能数量级 | 典型技能 |
|---|---|---|---|
| 初始（Starter） | 0 | 少量 | 基础攻击、火球术 |
| 新手（Novice） | 1 | ~5-8 / 学派 | 战吼、冰雹 |
| 中级（Adept） | 3 | ~3-5 / 学派 | 旋风斩、冰冻箭 |
| 大师（Master） | 5 | ~2-3 / 学派 | 流星雨、裂地 |
| 源力（Source） | 特殊获取 | ~1-2 / 学派 | 时间扭曲、神化 |

**门槛递增**：早期技能 1 点就能学，中期需要 3 点，后期需要 5 点——**投资越深，技能越强，但机会成本越高**（因为 10 个学派共享能力点）。

### 7.8 民用能力——战斗外的另一条成长线

| 类别 | 能力 | 效果 |
|---|---|---|
| **社交** | Persuasion 说服 | 对话选项成功率 |
| **潜行** | Sneaking 潜行 | 潜行速度+被发现距离 |
| **盗窃** | Thievery 盗窃 | 开锁+偷窃（每 NPC 仅限 1 次）|
| **鉴定** | Loremaster 鉴定 | 鉴定物品+查看敌人弱点 |
| **交易** | Bartering 交易 | 买卖价格 |
| **念力** | Telekinesis 念力 | 远距离移动物品范围 |

**设计哲学**：民用能力每级获得 1 点，和战斗能力点完全分开——**不存在"战斗强还是探索强"的二选一**。但民用能力点本身是稀缺的（~10 点分配给 6 项能力），所以团队内必须有分工。

### 7.9 升级点数分配

| 点数类型 | 每级获得 | 总量（满级 21）| 稀缺度 |
|---|---|---|---|
| 属性点 | 2 点 | ~40 点 | 中等 |
| 战斗能力点 | 1 点 | ~20 点 | **极度稀缺** |
| 民用能力点 | 1 点 | ~20 点 | 中等 |
| 天赋点 | 每 5 级 1 个 | 5 个 | **极度稀缺** |

**战斗能力点极度稀缺**是 DOS2 Build 系统的核心张力：
- 10 个学派争夺 ~20 点 → 不可能全修 → **必须取舍**
- 天赋只有 5 个 → 每个天赋必须"改变玩法"——Mnemonc（+3 槽）、Elemental Affinity（-1AP）、Executioner（击杀+2AP）等都是"点不点完全是两个游戏"级别的天赋

## 八、技能系统对项目的启发

### 8.1 直接借鉴

| DOS2 设计 | 我们可借鉴的方向 | 优先级 |
|---|---|---|
| **记忆槽 = 技能数量硬约束** | 我们的 JP 投入决定可学技能数——可以设计为"JP 总量有限，技能槽和专精度竞争" | ⭐⭐⭐ |
| **装备赋予技能（0 槽）** | 传说级武器/护甲自带技能，不占技能槽——让装备成为 Build 的一部分 | ⭐⭐⭐ |
| **学派门槛递增** | 我们的职业技能分 初级/中级/高级，需要对应 JP 等级——已有，验证正确 | ⭐⭐ |
| **物理/魔法双公式** | 我们的 Attack Type 三分法（Slash/Pierce/Crush）已有差异，但物理技能统一缩放于武器伤害这一点值得学习 | ⭐⭐ |
| **技能衍生链** | 某些技能施放后获得临时技能——我们的职业技能可以设计类似机制（如剑舞→3 段→每段独立 roll）| ⭐⭐ |
| **每技能每回合只能施放一次** | 简单有效的限制，防止同一强力技能连发 | ⭐⭐⭐ |

### 8.2 不借鉴

| DOS2 设计 | 不借鉴的原因 |
|---|---|
| **10 学派 + 每点 +5%** | 太碎片化，我们走"职业 × 武器"路线，不走"学派点数"路线 |
| **技能书合成** | 工艺系统复杂度高，与我们的"精简"哲学不符 |
| **Memory 属性** | 我们用 JP 投入代替，不需要单独属性——避免属性膨胀 |
| **源点系统** | 对我们来说太重，我们的"全力一击"已经类似但更简洁 |
| **民用能力** | 我们是纯战棋，不需要探索/交易/盗窃的民用系统 |

### 8.3 核心洞察——DOS2 技能系统的"一句话设计哲学"

> **"稀缺是趣味之母"**——能力点极度稀缺 → 必须取舍 → 取舍产生 Build → Build 产生重玩价值。

DOS2 的技能系统之所以深，不是因为技能多（200+），而是因为**你能同时用的只有 10-15 个**。每加一个技能都要考虑"值不值得占一个槽"。

映射到我们：**JP 的真正价值不是"解锁更多技能"，而是"让每个技能选择都有重量"**。如果 JP 足够让一个角色学完所有职业技能，那 JP 系统就失败了。

## 九、AP/CT 双轴对照表（与本项目的关键参考）

| 维度 | FFT (CT 派) | DOS2 (AP 派) | 战兄弟 (AP 派) | **我们（双轴并存）**|
|---|---|---|---|---|
| 行动条 | ✅ CT 累加 | ❌ Initiative 一次性排序 | ❌ 每轮 init 排 | ✅ CT 累加（学 FFT）|
| AP 系统 | ❌ 无 | ✅ 4-6 AP | ✅ 9 AP | ✅ **9 AP（保留 BB 路线）**|
| Wait 动作 | ✅ 扣 60 CT | ❌ 无 | ❌ 无 | 🟡 **建议加**（参考 FFT）|
| 元素互动 | 部分（地形） | ✅ 完整 | ❌ 无 | 🟡 Phase 3 候选 |
| 双护甲 | 单层 HP/MP | ✅ 物理/魔法 | ✅ 头/身 | ✅ 头/身（学 BB）|
| 多结局 | 多结局 | 多结局 | 单结局 | 🟡 Phase 3+ 多分支任务 |

**结论**：我们走的"CT (FFT) + AP (BB)"双轴路线是**两家工业方案的合并**——FFT 没 AP，DOS2 没 CT，**我们都有 = 创新点 + 风险点**。
