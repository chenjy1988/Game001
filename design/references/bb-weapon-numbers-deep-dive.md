## 零、我们 vs BB 的根本架构差异（阅读本报告前必读）⭐

在套用 BB 数据前，必须先明确我们项目和 BB 在**武器数据架构**上的 3 条根本差异——这决定了哪些 BB 数值可以直接抄、哪些必须做"翻译"才能落地。

### 差异 1：武器**无技能**，技能归属职业 ⭐

| 维度 | BB | 我们 |
|---|---|---|
| 武器是否自带技能 | ✅ 每件武器自带 1-3 个技能（如剑的 Slash/Riposte，斧的 Chop/Split Shield） | ❌ 武器**只有数据**（damage_base / AP / range / block / weight 等 7 字段） |
| 技能归属 | 武器持有者自动获得武器技能 | **职业 × 武器组合 + JP 投资**才能学到技能（如"斥候 + 匕首"才能学穿心刺） |
| 通用动作 | 武器自带"普攻"是默认动作 | 普攻 / 瞄头 / 全力一击 / 精准射击 是**通用攻击模式**（任何人会用，但受武器物理限制） |

**这意味着**：
- BB Wiki 武器表里"主技能 AP/疲劳"那一列**对我们武器属性无意义**
- 但这些技能数值作为**我们职业技能的设计锚点**仍然非常有用（见 § 五法则 3）
- 当我们看到"BB Fighting Axe 自带 Split Shield"，我们的翻译应是："拿斧的某个职业（押衙亲兵？跳荡？）通过 JP 学到 Split Shield"

### 差异 2：穿甲/渗透**不是武器属性**，由攻击类型统一决定 ⭐

| 维度 | BB | 我们 |
|---|---|---|
| 武器穿甲数据 | ✅ 每件武器单独配 `Ignores Armor` 和 `vs Armor%` 两个字段 | ❌ 武器**没有**这两个字段 |
| 穿甲来源 | 武器本身（即使同是斧，Hatchet 30% 无视 vs Pickaxe 50% 无视）| **由 Attack Type 三分法（Slash / Pierce / Crush）统一决定**（见 weapon-system.md 4.2）|
| 数据规模 | BB 每件武器 2 个穿甲数 × 50+ 武器 = 100+ 数字 | 我们 3 个攻击类型 × 2 个系数 = 6 个数字（玩家心智骤降）|

**这意味着**：
- BB Wiki 数值表里 `Ignores Armor` 和 `Vs. Armor` 两列**不能直接抄**
- 这两列的差异化作用，在我们项目里通过"武器选择哪种 Attack Type"来实现：
  - BB Warhammer 50% 无视 + 225% vs 护甲 → 我们体系下 = "Crush 武器（armor_mult=1.5 / hp_mult=0.7）"
  - BB Rondel Dagger 20% 无视 → 我们体系下 = "Pierce 武器（armor_mult=0.7 / 渗透 0.15）"
- **后果**：BB 同一类武器（如 Maces）内部的"白绿蓝红"穿甲分化，我们体系不能复刻——我们的差异化在"职业精通系数"和"传说词条"层而非武器基础数据层

### 差异 3：耐久度**有但暂未深入设计** ⚠️

| 维度 | BB | 我们 |
|---|---|---|
| 耐久度字段 | ✅ 每件武器有耐久度（24-120 范围）| ✅ 已有字段，但**详细机制 Phase 4 才设计** |
| 耐久衰减规则 | 命中 / 被命中 / Split Shield 等都消耗耐久 | 待定 |
| 修理 | 城镇铁匠 / 出战间隙 | 待定（铁匠后勤系统 Phase 4）|
| 红装词条 | 耐久 ×0.9~1.4 是命名武器**始终修改**的属性 | 待定 |

**这意味着**：
- BB 耐久度作为"武器档位辅助标尺"（贵的武器普遍耐久更高）的设计**可以保留参考**
- 但**当下不能把"耐久度差异"作为武器差异化的卖点**——耐久机制不到位，玩家感知不到差异
- 报告里 § 二、§ 三的 "耐久" 列暂时仅作"BB 数据快照"，**不进入数值锚点表**

### 校准后的"BB 数据可用性"清单

| BB 数据列 | 是否可直接抄 | 我们的对应 |
|---|---|---|
| 价值 / Worth | 🟢 可参考价格档位曲线 | 经济系统锚点 |
| **伤害（Damage）** | 🟢 **取中位数 × 我们体系做映射** | damage_base 的核心锚点 ⭐ |
| 无视护甲 (Ignores Armor) | ❌ **不抄** | 由 Attack Type 决定 |
| Vs. 护甲 % | ❌ **不抄** | 由 Attack Type 的 armor_mult 决定 |
| 耐久度 | 🟡 仅作档位参考 | Phase 4 详定 |
| **疲劳惩罚 (Max. Fatigue)** | 🟢 **直接对齐** | 我们 weight 的锚点 ⭐ |
| 武器主技能 AP | 🟢 作为**职业技能**的 AP 锚点 | 4 AP 单手 / 6 AP 双手 |
| 武器主技能疲劳 | 🟢 作为**职业技能**的隐藏疲劳锚点 | 13/25 两档 |
| 头部命中加成 | 🟡 部分参考 | 我们用 1h 25% / 2h 15% 二分 |
| 盾伤 (Shield Damage) | 🟢 作为**反盾职业技能**数值锚点 | "斧 Split Shield 8/12/16/30 盾伤"等 |

→ **核心结论**：BB 数据的"价格 / 伤害 / 疲劳惩罚 / 主技能 AP-疲劳 / 盾伤" 这 5 列可用，**穿甲两列不抄，耐久仅参考**。

## 一、BB 武器分类总览（8 大近战类 + 4 大远程类）

```
近战 8 类：
  ├─ Daggers 匕首（1h，仅穿刺，专破 HP 流派）
  ├─ Swords 剑（1h/2h，全能均衡，命中 +5~10%）
  ├─ Axes 斧（1h/2h，砍 + 破盾）
  ├─ Maces 钉头锤（1h/2h，钝 + 控制晕眩）
  ├─ Hammers 战锤（1h/2h，纯破甲）
  ├─ Cleavers 砍刀 / 鞭（1h/2h，流血专精）
  ├─ Flails 连枷（1h/2h，无视盾防）
  ├─ Spears 枪（1h/2h，命中 +20% / 矛墙）
  └─ Polearms 长柄武器（仅 2h，2 格距离）

远程 4 类：
  ├─ Bows 弓（射程 7，AP 4，疲劳 15-20）
  ├─ Crossbows 弩（射程 6，AP 3，疲劳仅 5，但要 4 AP 上弦）
  ├─ Firearms 火器（射程 2，AP 3，命中无视掩体，9 AP 装填）
  └─ Throwing 投掷（射程 4，消耗品，5/6 发）
```

每一大类都有**明确专攻 + 明确弱点**——没有任何一类是"全能武器"。这是 BB 的第一设计法则。

  - 弓 7 格 / 弩 6 格 / 投掷 4 格 / 火器 2 格
  - 投掷武器射程短但伤害高，强迫玩家"扔完就上"

## 四、武器精通（Mastery）—— 反向印证武器定位的钥匙 ⭐

> Mastery 系统反向定义了每类武器的"真正核心"——精通修改的部分，就是 BB 设计师认为这类武器最该被强化的部分。这是**理解 BB 武器哲学最重要的窗口**。

| 武器精通 | 通用减疲劳 | 专属强化（揭示设计意图）|
|---|---|---|
| **Sword** | -25% | Riposte 无命中惩罚 / Split + Swing +10% 命中 / Gash 创伤阈值 -50% | → 防守反击 + AoE + 制造伤势 |
| **Axe** | -25% | Split Shield 盾伤 +50% / Round Swing +5% 命中 | → **斧 = 反盾 + AoE** |
| **Mace** | -25% | 所有击晕几率提升至 **100%** | → **锤 = 100% 控场** |
| **Hammer** | -25% | Destroy/Demolish Armor 对甲 +33% / Shatter +5% 命中 | → **战锤 = 极致破甲** |
| **Spear** | -25% | Spearwall 突破后不再失效 / 可相邻激活 | → **枪 = 极致防御** |
| **Polearm** | -25% | 技能 AP 从 6 降至 **5** / 无相邻惩罚 | → **长柄 = 二线高效** |
| **Dagger** | -25% | 所有刺击类技能 AP 从 **4 降至 3**（每回合多攻击 1 次） | → **匕首 = 高频穿心** |
| **Cleaver** | -25% | 流血伤害**翻倍**（5/r → 10/r） / Disarm 惩罚减半 | → **砍刀 = 持续流血** |
| **Flail** | -25% | Lash/Hail 无视盾防 / Pound 头部 +10% 伤害 | → **连枷 = 反盾 + 强制爆头** |
| **Bow** | -25% | 视野 +1 / 射程 +1 | → **弓 = 视野与距离** |
| **Crossbow/Firearms** | -25% | 弩额外 20% 无视护甲 / 火器装填 9→6 AP | → **弩 = 强化破甲** |
| **Throwing** | -25% | 2 格 +30% / 3 格 +20% 伤害 | → **投掷 = 近距爆发** |

**精通系统的设计教科书价值**：
1. **每个精通只强化"核心 1-2 件事"**——不是堆数值
2. **AP/疲劳减免是通用基线**（-25% 疲劳是所有精通共通的"基础税"）
3. **专属效果都是"原本就在做的事，但更极致"**：
   - Mace 本来就有击晕几率 → 精通后 100%
   - Cleaver 本来就有流血 → 精通后翻倍
   - Dagger 本来就专精 HP 直伤 → 精通后再加 1 次/回合
4. **从不出现"+X% 伤害"**——精通强化机制而不是数字

→ **可直接抄到我们项目的设计法则**：未来"职业精通"时不要写"+20% 伤害"，要写"使该武器最具特色的机制更极致"。

## 五、BB 武器数值的 5 大底层设计法则（可直接落地）

### 法则 1：**双轴定位法**（在我们体系内的翻译）

**BB 原始法则**：每类武器在"对 HP × 对甲"两轴上抢一个象限，靠**逐件武器配 `Ignores Armor` 和 `vs Armor%`** 实现。

**关键校准（差异 2）**：我们武器**没有**穿甲属性，穿甲由 Attack Type 三分法统一决定。所以"对 HP × 对甲"双轴在我们体系里**不在武器层而在攻击类型层**：

```
我们体系下 BB 武器双轴的翻译：

  破甲方向（armor_mult 高）
        ▲
        │
   Crush 1.5 ●（战锤、单手锤、长棍）
        │
        │
        ●─────●  Slash 1.0（基线：剑、横刀、斧）
        │
        │
        ●  Pierce 0.7（匕首、矛、弓、弩）—— 但有 0.15 渗透补偿
        ▼
  破甲方向低
```

**法则改写后给我们的启示**：
- BB 的"武器内部分化"（如 Hatchet 30% 无视 vs Warhammer 50% 无视）—— **我们做不到，也不应该做**（违反差异 2）
- 我们的"破甲倾向差异化"由两个杠杆控制：
  1. **Attack Type**（Slash/Pierce/Crush）= 武器选择的"破甲哲学"
  2. **职业技能**（破甲爆发 / 流血 / 击晕）= 真正的差异化
- → 所以"Hammer 极破甲 vs Mace 控场"在我们体系的实现 = **同是 Crush 武器，但解锁的职业技能不同**：
  - 战锤+悍卒 → "锐不可当"忽视格挡
  - 钉头锤+押衙亲兵 → "击晕"（待 Phase 3 设计）
- **武器不背"破甲数值差异"的锅**，由攻击类型和职业技能背

**对 weapon-system.md 现有设计的反向印证**：
- 4.2 三攻击类型 × 2 系数 = 6 个数字（替代 BB 的 100+ 武器穿甲数据）→ ✅ 这个简化方向是正确的
- 4.7 暴击设计（全局暴击只加伤害；匕首等专属流派另行强化 HP 渗透）→ ✅ 这才是我们体系的"破甲特色"载体

### 法则 2：**AP/疲劳惩罚 = 武器档位标尺**

BB 用"疲劳惩罚"作为档位锚点，比"伤害"更稳定：

| 档位 | 1h 疲劳惩罚 | 2h 疲劳惩罚 | 价格区间 |
|---|---|---|---|
| 入门（白装）| -6 ~ -8 | -10 ~ -14 | 100-500g |
| 标准（中级）| -10 | -14 ~ -16 | 1000-2000g |
| 精英（红装级别）| -12 | -16 ~ -18 | 2000-3500g |
| 兽人/极重 | -18 ~ -22 | -30 ~ -34 | 1100-2000g（但难用）|

**对我们项目的映射**：
- 我们的 weapon `weight` 字段（横刀 4 / 陌刀 9）已经在做相同事情
- 建议把 weight 显式分档：1（匕首）/ 3（轻 1h）/ 4-5（标准 1h）/ 6-9（2h）
- 兽人风格在我们项目可作为传说级"重型"词条而非独立武器

### 法则 3：**BB 武器技能 = 我们职业技能的数值锚点** ⭐

**关键校准（差异 1）**：BB 的武器技能在我们体系里**不能存在于武器属性**，但可以作为"职业 × 武器组合解锁的职业技能"的**数值锚点**。

**BB 武器技能数值速查表（作为我们职业技能设计的参考库）**：

| BB 武器系 | BB 普攻技能 | BB 特色技能 | 翻译到我们体系 |
|---|---|---|---|
| 1h 斧 | Chop 4/13 | **Split Shield 4/15**（破盾） | 押衙亲兵/跳荡 + 单手战斧 → 学"裂盾"职业技能 |
| 1h 锤 | Bash 4/13 | **Knock Out 4/25**（75% 眩晕） | 押衙亲兵/虞候 + 单手战锤 → 学"击晕"职业技能 |
| 1h 战锤 | Hammer 4/14 | **Destroy Armor 4/25**（+50% 破甲） | 悍卒 + 单手战锤 → 学"碎甲"职业技能 |
| 1h 砍刀（弯刀）| Cleave 4/12（自带流血）| **Decapitate 4/20**（斩首爆发） | 不良人 + 弯刀 → 学"枭首"职业技能 |
| 1h 矛 | Thrust 4/10 | **Spearwall 4/30**（反冲锋） | 跳荡/枪手 + 短矛 → 学"架枪反骑"职业技能（与 weapon-system 4.9.3 已有对齐 ✅）|
| 1h 连枷 | Flail 4/13（无视盾）| **Lash 4/25**（强制头部） | （未来武器扩展时考虑加入）|
| 1h 匕首 | Stab 4/7 | **Puncture 4/20**（100% 直伤） | 斥候/不良人 + 匕首 → 学"穿心刺"（与 weapon-system 4.9.2 已设计 ✅）|
| 2h 大剑 | Slash 4/13 | **Overhead Strike 6/15**（+20 伤）/ Split 6/30 | 陌刀手 + 陌刀 → "全力一击" / "横扫千军"（已有 ✅）|
| 2h 大斧 | — | **Round Swing 6/35**（6 人 AoE） | 悍卒 + 双手战斧 → 学"横扫"或类似 AoE 职业技能 |
| 2h 双手锤 | — | **Smite 6/15** + **Shatter 6/30**（33% 击退）| 悍卒/铁骑 + 双手战锤 → 学"重击"+"震慑" |

**法则改写**：BB 武器技能不能直接抄，但其 **AP/疲劳数值刚好是我们职业技能的设计标尺**：

| 职业技能档位 | AP | 隐藏疲劳 | BB 同档技能参考 |
|---|---|---|---|
| 初级（任何阶段可学）| base + 1 | +5~7 | Stab 4/7 / Cleave 4/12 |
| 中级（精通解锁）| base + 1~2 | +10~15 | Decapitate 4/20 / Spearwall 4/30 / Knock Out 4/25 |
| 高级（精通+JP重投）| base + 2 | +15~25 | Strike Down 6/30 / Round Swing 6/35 |

→ **当下行动**：把 weapon-system.md 4.9.3 节的职业技能表，按此锚点表逐个校准 AP/疲劳数值。

### 法则 4：**疲劳的"一长一短"节奏锚点**

校准后的法则（剔除"武器技能"部分，只保留普攻和职业技能的疲劳节奏）：

```
我们体系下的疲劳节奏锚点：

普攻（任何武器）：
  AP = 武器 base_ap（1h: 4 / 2h: 6-7 / 弓: 4 / 弩: 6）
  疲劳 = ceil(base_stamina × weight_mult)，约相当于 BB 的 10-15
  → 一回合可发动 2 次普攻（1h）或 1 次（2h）

通用攻击模式（瞄头 / 全力一击 / 精准射击）：
  AP = base + 1~2
  隐藏疲劳 = +8~12（建议从当前 +5~8 上调）
  → BB 锚点：Aimed Shot 7/20、Overhead Strike 6/15

职业技能（中级）：
  AP = base + 1~2
  隐藏疲劳 = +10~15
  → BB 锚点：Spearwall 4/30、Knock Out 4/25、Decapitate 4/20

职业技能（高级 / 大招）：
  AP = base + 2
  隐藏疲劳 = +15~25
  → BB 锚点：Round Swing 6/35、Strike Down 6/30
```

**我们当前疲劳值偏低的诊断**：
- weapon-system.md Step 9 给的是：瞄头+8、全力+5、穿心刺+5
- 与 BB 锚点比较：BB 用 20-30 疲劳的强力技能营造"用一次想三次"
- **建议调整**：瞄头 +8→+10、全力 +5→+8、穿心刺 +5→+12（与 BB Puncture 20 对应）
- → 这样玩家用 5 次穿心刺 ≈ 60 疲劳，必须穿插休息或普攻，"赌一刺破甲"的紧张感才出得来

### 法则 5：**克制目标由"武器 × 职业技能"组合定义**（差异 1 的体现）

**BB 原始法则**：每类武器自带克制技能（斧自带 Split Shield、锤自带 Knock Out），所以"武器即克制位"。

**校准后我们的法则**：武器只决定**物理可能性**（Slash/Pierce/Crush + 单/双手 + 距离），**真正的克制位由"哪个职业拿这把武器 + 学了什么职业技能"决定**。

| 克制位 | BB 实现方式 | 我们的实现方式（武器 + 职业技能） |
|---|---|---|
| 反重甲 | 战锤（武器自带 Destroy Armor） | **战锤 (Crush) + 悍卒/押衙的"碎甲"职业技能** |
| 反盾 | 斧（武器自带 Split Shield） | **战斧 (Slash) + 跳荡/押衙的"裂盾"职业技能** |
| 反盾（机制流） | 连枷（武器自带无视盾防） | （未来扩展时再说，连枷不在 Phase 2 范围）|
| 反硬目标 | 匕首（武器自带 Puncture） | **匕首 (Pierce) + 斥候/不良人的"穿心刺"职业技能** ✅ 已设计 |
| 反无甲 | 砍刀（武器自带 Cleave 流血） | **弯刀 (Slash) + 不良人的"枭首"职业技能** |
| 反冲锋 | 矛（武器自带 Spearwall） | **长矛 (Pierce) + 跳荡/枪手的"架枪反骑"职业技能** ✅ 已设计 |
| 控场 | 锤（武器自带 Knock Out） | **钉头锤 (Crush) + 押衙的"击晕"职业技能** |
| 远程破甲 | 弩（武器自带 50% 无视护甲）| **弩 (Pierce) + 弩手的"抵近射击"职业技能** ✅ 已设计 |

**对我们项目的关键启示（经差异 1 校准后）**：

> **不要给武器配"克制目标"。给"武器 + 职业 + 技能" 三元组配克制目标。**

这意味着：
- ❌ 错误："单手战斧的克制目标是盾兵"
- ✅ 正确："**跳荡 + 单手战斧 + 裂盾职业技能** 的组合克制盾兵"
- ❌ 错误："匕首是反硬目标的"
- ✅ 正确："**斥候/不良人 + 匕首 + 穿心刺** 的组合反硬目标"

**所以"补武器"不是当下的最优解，"补职业技能"才是**：

| 缺位 | 不需要补武器 | 补什么职业技能 | 装备到哪个职业 × 武器 |
|---|---|---|---|
| 反盾 | ❌ 已有单手战斧 / 单手战锤 | 加"裂盾"职业技能（4 AP / 15 疲劳 / 直接对盾 16 伤）| 跳荡 / 押衙 + 单手战斧 |
| 控场（眩晕） | ❌ 已有单手战锤（Crush） | 加"击晕"职业技能（4 AP / 25 疲劳 / 75% 1 回合眩晕） | 押衙 + 单手战锤 |
| 流血 | ❌ 已有横刀 / 剑（Slash） | 加"枭首"或"放血"职业技能（4 AP / 20 疲劳 / 流血 5 HP × 2 回合）| 不良人 + 匕首 / 弯刀 |
| 远程破甲 | ✅ 已有擘张弩 | "抵近射击"已设计 ✅ | 弩手 + 擘张弩 |

→ **结论**：我们的武器原型清单（5.1-5.3 节）从生态位角度看 **已经够用，缺的是职业技能而非武器**。

→ 这反过来印证了我们设计哲学的优越性：**武器极简 + 职业技能丰富 = 武器表稳定不膨胀，差异化全在职业层叠加**。

## 七、给数值策划的"可直接套用"行动清单

### 🎯 优先级 P0：调整 2h 武器伤害锚点

**问题**：当前 2h/1h = 1.44×，BB 行业标准 = 1.75×，使 Double Grip 的取值进退两难。

**行动**：把 2h 武器 damage_base 上调到 BB 比例：

| 武器 | 当前 | 建议 | 调幅 | 锚点参考 |
|---|---|---|---|---|
| 长矛 | 45 | **55** | +22% | Spetum 65 |
| 双手战斧 | 65 | **80** | +23% | Greataxe 90 |
| 双手战锤 | 55 | **70** | +27% | 2H Hammer 75 |
| 陌刀 | 60 | **75** | +25% | Greatsword 92.5 |

调整后 2h/1h ≈ 1.7×，**Double Grip 取 +25% 即可（BB 原版值）**。

### 🎯 优先级 P1：补足"职业技能"（而非武器）

校准后正确的做法 ⭐：**不补武器，补职业技能**（基于差异 1 + 法则 5 的结论）。

我们的武器原型已覆盖所有物理可能性，缺的是 3 个克制位的职业技能：

| 缺位 | 职业技能名 | 装备到 | AP / 疲劳 | 效果 | BB 锚点 |
|---|---|---|---|---|---|
| 反盾 | **裂盾** | 跳荡/押衙 + 单手战斧 | 4 / 15 | 直接对盾耐久 16 伤 | Split Shield |
| 控场 | **击晕** | 押衙 + 单手战锤 | 4 / 25 | 75% 概率晕 1 回合，目标 +10 疲劳 | Knock Out |
| 流血 | **放血** | 不良人 + 匕首/弯刀 | 4 / 20 | 命中后流血 5 HP × 2 回合 | Decapitate / Cleave |
| （可选）破甲爆发 | **碎甲** | 悍卒 + 单手战锤 | 4 / 25 | armor_mult 临时提升至 2.0 | Destroy Armor |
| （可选）AoE | **横扫** | 悍卒 + 双手战斧 | 6 / 35 | 命中前方 3 格敌人，命中-15% | Round Swing |

**详细设计请放到 `class-system.md` 而非 `weapon-system.md`**——这本身就是差异 1 的落地。

### 🎯 优先级 P2：把"通用攻击模式"的疲劳值校准到 BB 锚点

| 模式 | 当前隐藏疲劳 | BB 同类技能 | BB 疲劳 | 建议 |
|---|---|---|---|---|
| 普攻 | base × weight_mult | Bash/Chop | 12-14 | ✅ 类似 |
| 瞄头 | +8 | Aimed Shot（弓） | 20 | 调到 +10~12 |
| 全力一击 | +5 | Overhead Strike / Cudgel | 15 | 调到 +8 |
| 穿心刺 | +5 | Puncture | **20** | 调到 +12~15 |

**结论**：我们的隐藏疲劳偏低，BB 用 20-30 疲劳的强力技能营造"用一次想三次"的节奏，我们当前 5-8 略浅。

### 🎯 优先级 P3：传说武器的"双轴爆发"模板（按差异 2 校准）

BB 命名武器走"6 选 2 抽签"。**校准后剔除我们体系不存在的轴**（破甲、直伤——这些是 BB 武器穿甲属性的扩展，我们没有）：

```
我们体系下传说武器加成可用的轴（取代 BB 6 选 2）：

输出轴：
  - 伤害 ×1.05~1.30          ← 对应 BB 伤害 ×
  - 暴击率 +5~15             ← 对应 BB 爆头率（我们用暴击替代）

操作轴：
  - 攻击 AP -1               ← 对应 BB 技能疲劳减免
  - 重量 -1~2                ← 我们独有（影响 weight_mult）
  - 命中 +5~10               ← 我们独有

防御轴：
  - 格挡 +3~10               ← 对应 BB 盾伤逆思路
  - 受击时 X% 减伤            ← 我们独有

特色轴（双向必带）：
  - HP < 50% 时伤害 +20% / HP > 80% 时命中 -10
  - 第一次攻击 +30% / 后续 -10%
  - ...

负向代价（必须配 1 个）：
  - 自损 X HP / 命中 -X / 移动 AP +1 / 重量 +1 等
```

**校准后的传说武器抽签规则**：
- 主轴（必选 1 个，从输出 / 操作 / 防御 三系任选）
- 副轴（必选 1 个，**必须是与主轴不同的系**——强制跨系组合）
- 负向代价（必选 1 个，配合双向词条设计哲学）

→ **数值红线（与 weapon-system.md 8.4 节同步）**：
- 单件武器正向词条加成总和 ≤ +30% 武器伤害
- 负向代价不计入红线
- "破甲" / "直伤" 不进入词条池——这些由 Attack Type 统一管，避免武器层和攻击类型层冲突

## 八、BB 武器系统对我们 design.md "战斗模型 v3" 的核心启示

| 设计决策 | 对照 BB | 我们的状态 | 建议 |
|---|---|---|---|
| **武器技能归属（差异 1）** | 武器自带技能 | **职业 × 武器组合 + JP 解锁** | ✅ **保持，更深度** |
| **穿甲属性（差异 2）** | 武器逐件配 `Ignores Armor` + `vs Armor%` | **由 Attack Type 三分法统一决定** | ✅ **保持，玩家心智更轻** |
| **耐久度（差异 3）** | 武器档位辅助标尺，命名武器固定 ×0.9~1.4 | 已有字段，**Phase 4 详定** | 当下不作差异化卖点 |
| 三攻击类型（Slash/Pierce/Crush）| BB 用穿甲两列实现（武器层）| 我们用 armor_mult/hp_mult 系数（攻击类型层）| ✅ 已对齐 |
| 武器多模式（剑/陌刀双攻）| BB 没有 | 玩家临场选 Slash 或 Pierce | ✅ 更优雅，保持 |
| 取消伤害上下限 | BB 用 min-max | 我们用 base + 气力档位 | ✅ 更利于玩家预判 |
| 武器精通 = 数值 +20% | BB 精通 = 机制强化（不直接 +伤害）| 我们走 +20% 路线 | **建议改：精通 +5~10% + 解锁特色机制** |
| 反割草 cap | BB 没有 | 我们有 base × 2.0 | ✅ 更稳，保持 |
| 头部命中 25%/15% | BB 全武器 25%（部分 +5%）| 我们用 1h/2h 区分 | ✅ 更精致，保持 |
| 双手握持 +25% | BB +25% | 我们 +20%~25%（待定）| **取 +25% 锚定到 BB 标准** |
| 词条总和 ≤ +30% | BB 命名武器无明确上限（实际 ~+30~50%）| 我们更克制 | 建议传说级允许 +35%（含负向词条则不限）|
