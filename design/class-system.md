> 核心目标：18 个战斗职业差异分明，6 个后勤职业撑起战场外深度，JP 系统驱动 build 探索。

## 一、设计哲学（不可摇摆）

### 4 条核心原则

1. **全员战斗员**——没有"脆皮远程"或"纯辅助"职业，连弩手都是中重甲+横刀
2. **远程是装备不是职业**——步弓人人可装，准头看 `melee × 0.6 + bow_mastery`
3. **辅助下放为通用技能池**——任何职业都能学辅助技能，但有擅长域
4. **JP 驱动成长，非等级驱动**——职业深度来自 JP 投资，不是刷级

### 红线约束

- ❌ 任何战斗职业都不能在战斗内回 HP（坚守减法制）
- ❌ 不能存在"打不动人"的辅助型上场单位
- ❌ 后勤 NPC 永远不能上场战斗
- ✅ 单职业被动增伤总和 ≤ +30% 武器伤害（反割草红线 #4）
- ✅ 辅助型职业 HP 介于刺客（70-80）和重甲（120-150）之间，能近战自卫
- ✅ 强力辅助技能必须有"全队限 1 次"或"高 AP 消耗"约束

## 二、双层架构总览

```
战场内：18 战斗职业 + 通用辅助技能池 → BB 哲学（每人都参战）
战场外：6 个纯辅助职业（NPC 雇佣，占小队名额）→ FFT/Berwick 灵魂（生活/经营）
```

**设计意图**：
- BB 哲学要求"每个上场单位都是战斗员"——纯辅助坐板凳违反小队哲学
- FFT 灵魂要求"职业身份感 + 多样性"——但纯辅助职业天然破坏减法制和永久死亡
- **解法**：辅助能力分三层下放
  1. **战斗职业**（18 个）→ 全员战斗员，每人能近战 / 远程自卫
  2. **通用辅助技能池**（5 大类）→ 任何职业都能学的辅助技能，但每个职业有 3 类擅长域
  3. **后勤职业**（6 个 NPC）→ 不上场，深度联动生活系统

### 3.5 骑兵战术定位：游击、冲锋与冲击阵型

骑兵不是“Move 更高的步兵”，而是通过**路线、距离、阵型和反制**形成独立战术层。轻骑与铁骑分工必须清晰，否则会互相抢定位。

#### 轻骑 —— 游击骚扰 / 残敌收割

**核心身份**：不和完整阵线硬拼，靠高 Init、骑射、侧翼位移制造压力，等敌人阵型被拉开后收割残血。

- **游击骚扰**：移动后射击、射击后小幅撤离，持续压低敌方 Init / morale / 站位质量。
- **拉扯阵型**：逼迫敌方转向、分兵或提前消耗 AP，不以单次伤害取胜。
- **残敌收割**：优先击杀低 HP、脱离护卫、已破甲或士气动摇的目标；对残血目标获得伤害加成，靠机动脱离 ZoC。
- **风险边界**：被长矛架枪、森林/泥地、ZoC 包夹克制；轻骑冲进完整枪阵应当是高风险行为。

**典型回合感受**：
1. 沿侧翼移动 2-3 格，骑射压制后撤 1 格。
2. 等前排打散阵线后，切横刀突入击杀残血。
3. 击杀后不站桩，用剩余 AP 或技能离开敌方主力反击范围。

#### 铁骑（具装）—— 正面冲锋 / 冲击阵型核心

**核心身份**：昂贵、迟缓、正面爆发强。铁骑不是持续输出最高的单位，而是打开战线缺口的“战术锤”。

- **正面冲锋**：需要直线/近直线助跑距离；距离越足，冲击越强。
- **破阵目标**：优先撞开盾墙、重步兵、敌方关键单位周围的护卫，而不是追逐散兵。
- **首击压制**：对满血/阵型完整目标获得额外伤害，用来体现重骑第一轮冲击的破阵价值。
- **冲后代价**：冲锋后 Init 低、气力消耗高、转向困难；若没打穿阵线，容易被围住。
- **明确反制**：枪兵架枪、地形阻滞、拒马/障碍、侧翼包抄都应显著削弱铁骑冲锋。

#### 冲击阵型

**定位**：2 名以上铁骑/重骑保持相邻或平行推进时形成的短时阵型，用于正面撕开敌方阵线。

**建议规则锚点（Phase 2.5+）**：

| 规则点 | 设计 |
|---|---|
| 成型条件 | 至少 2 名骑兵相邻，面向同一大方向，且本回合开始时未被 ZoC 缠住 |
| 发动方式 | 选择“冲击阵型”后，阵型成员按顺序冲锋；首骑破防，后骑扩大战果 |
| 收益 | 冲锋命中 +10%，对盾/重甲目标护甲伤害 +20%，命中后击退 1 格 |
| 代价 | 每名骑兵额外消耗 15-25 气力；冲锋后本回合 Defense -10 |
| 反制 | 架枪单位可打断首骑；障碍/森林/泥地使阵型失效；被侧面 ZoC 接触会拆散阵型 |

**设计红线**：
- 冲击阵型是“开战线”的战术按钮，不是稳定清场工具。
- 不能无视枪兵体系。枪兵必须是骑兵正面冲锋的硬反制之一。
- 冲锋收益主要体现在破阵、击退、破甲和士气压迫，不应简单做成纯伤害倍率堆叠。

### 3.6 斥候核心价值：侦察决定战争形态

**设计哲学**：斥候不是弱化版游侠，而是**决定战斗是"主动出击"还是"被动挨打"的关键角色**。没有斥候的队伍走进战场是"盲人摸象"，有斥候的队伍是"知己知彼"。

**斥候侦察影响战场四要素**：

| 要素 | 无斥候 | 有斥候 | 有斥候 + Wisdom 高 |
|---|---|---|---|
| **战前布局** | 随机出生点 | 可选择 1-2 个出生区域 | 可选择任意出生区域 |
| **敌军情报** | 仅看到视野内敌人 | 获得敌军总数范围（±3）| 精确敌军数量 + 1-2 个精英兵种信息 |
| **战场视野** | 默认视野半径 5 | 视野半径 5 + 斥候额外侦察范围 | 视野半径 5 + 远视技能可扩展到 8-10 |
| **伏击预警** | 被伏击无预警 | 被伏击前 1 回合预警 | 提前发现伏击点 + 部分揭示伏击单位 |

**Wisdom 对侦察的加成公式**：
```
侦察精度 = base_accuracy + wisdom × 0.5
敌军数量误差 = max(1, floor(5 - wisdom / 20))   // Wisdom 20→误差4, Wisdom 80→误差1, Wisdom 100→精确
出生点选择数 = 1 + floor(wisdom / 40)             // Wisdom 40→2选, Wisdom 80→3选
伏击发现概率 = 30% + wisdom × 0.5%               // Wisdom 60→60%, Wisdom 100→80%
```

**核心设计意图**：
- **斥候让 Wisdom 属性不再是"方术专属"**——高 Wisdom 斥候 = 最强情报官
- **侦察结果影响战斗决策**：知道敌方有重甲精锐 → 提前配穿甲武器；知道敌军数量少 → 主动出击
- **斥候不是"必带"而是"强烈推荐"**——没有斥候也能打，但会失去战场主动权
- **上级职业延续侦察**：游奕都巡官继承斥候侦察能力 + 骑乘机动，不良人继承斥候潜伏 + 暗杀

**与 design.md 的关联**：战前侦察结果由军师（后勤 NPC）提供大地图层面的情报（敌方阵容类型），斥候在战斗开始前提供战术层面的情报（出生点、敌军数量、伏击点）。二者叠加但不重复。

### 4.3 Wisdom 在战斗中的作用 ⭐

**设计哲学**：
> Wisdom 不只是斥候侦察的属性，也是**高速 build、暴击与战术夹击的核心成长属性**。让 Wisdom 真正进入战斗循环，避免"半残属性"的尴尬。

#### 4.3.1 已落地的作用

| 作用 | 公式 | 触发场景 |
|---|---|---|
| **战前侦察** | 见 § 3.6 | 斥候 / 游奕都巡官 / 不良人 |
| **暴击率加成** ⭐ | `crit_chance += max(0, wisdom - 40) × 0.2` | **所有战斗职业** |
| **夹击触发率** ⭐ | 正面格队友 A 主攻击命中后判定：`pincer_chance = clamp(20% + (A.wisdom - T.wisdom) × 0.5%, 5%, 75%)`；背后格队友 B 补刀 | **所有近战职业（正面位发起）** |

**暴击率梯度**：

| Wisdom | 加成 | 典型职业 |
|---|---|---|
| ≤ 40 | 0 | 陌刀手 / 悍卒 / 铁骑（重武器职业 Wisdom 天然低）|
| 50 | +2 | 中位职业 |
| 60-70 | +4~6 | 不良人养成中期 |
| 80 | **+8** | 斥候 / 诗剑仙 / 方士战兵基线 |
| 100 | **+12** | 神装 + 满养成顶配 |

→ 暴击率软上限 50%（详见 weapon-system.md § 4.7）

**夹击概率示例**（正面格主攻击命中后）：

| Wisdom 差值（攻-守） | 夹击触发率 |
|---|---|
| -20 | 10% |
| 0 | 20% |
| +20 | 30% |
| +60 | 50% |
| +110 | 75%（封顶） |

#### 4.3.2 设计意图

1. **重武器职业 Wisdom 天然低 → 不亏也不抢特色**——陌刀手 Wisdom 30 时，公式产出 0 加成，靠 base damage 高 + 全力一击爆发
2. **Wisdom 系职业（斥候/诗剑仙/方士战兵/不良人）真正受益**——核心爆发是多次独立暴击 roll，Wisdom 提升 8-12% 暴击率在长线战斗中显著放大爆发期望
3. **僧兵的 Wisdom 双向收益**——既驱动暴击（普攻多次），又解锁后续"念力护体"等专属能力（Phase 3）
4. **高 Wisdom 近战的前压包夹收益被显式放大**——正面队友堵路发起攻击、背后队友补刀；需阵型配合，不是平推堆数值（详见 design.md §九）
5. **远程职业自然吃到红利**——弩手/轻骑/蕃骑 Wisdom 中等（40-55），获得 0~3 暴击加成，符合"远程也需要观察判断"的物理直觉
6. **旧艺负载（兼修带宽）**——Wisdom 决定跨职继承技能能带多少；**本职零负载**，低 Wisdom 猛将仍能用满本职树（§5.3.2）

#### 4.3.3 旧艺负载（兼修带宽）⭐

> **记录日期**：2026-06-11 · 与 §5.3.2 v0.3 同步。

| 概念 | 规则 |
|---|---|
| **本职记忆** | 当前 `job_id` 已学技能 → **Weight = 0**，不占 Wisdom |
| **旧艺负载** | 从 **其他** `job_id` 继承并 **激活** 的技能 → `Σ weight ≤ wisdom` |
| **设计意图** | Wisdom 高 = 多职 hybrid 奇才；Wisdom 低 = 本职肌肉记忆不受影响，兼修只能带少量轻技 |
| **转职** | 旧艺 **默认卸载**；本职 + 池内武器专精 + 通用 **保持**（§5.3.2） |

**带宽速查**（Wisdom 30 vs 120，仅兼修）：

| 档次 | Weight | W30（莽夫） | W120（奇才） |
|---|---|---|---|
| T1 轻量 | 10–15 | ~2 个 | ~8–12 个 |
| T2 中量 | 20–30 | ~1 个 | ~4–6 个 |
| T3 重量 | 40–50 | ❌ | ~2–3 个 |
| T4 终极 | 60+ | ❌ | ~1–2 个 |

→ 分级表与示例见 §5.3.2；实现字段 `Ability.weight`、`Unit.inherited_skills` 见 §十二。

### 4.4 气力对战斗属性的影响

**设计原则**：气力首先影响动作质量（攻击命中率、防御稳定性），其次才影响输出倍率。气力不足不锁普通行动，但会让角色更容易打空、更难防住，并且伤害打折。

| 剩余气力比例 | 状态 | 命中修正 | 防御修正 | 伤害系数 |
|---|---|---|---|---|
| > 50% | 精力充沛 | 0 | 0 | **1.0** |
| 20%-50% | 疲劳 | -10% | Defense -5 | random(0.80, 0.90) |
| 0%-20% | 低气力 / 力竭 | -20% | Defense -10 | random(0.70, 0.80) |

> 气力三档为正式 debuff（`精力充沛` / `疲劳` / `力竭`），由 `CombatModifier` 按剩余气力比例切换，经 `Unit.get_active_debuffs()` 汇总并与「手拙」「动摇」等叠加；**会在 UI 状态栏外显**。

### 4.5 性别与属性天赋（成长向）

> **Phase 3 / K2** 实装；与 §七 **背景**、[`personality-system.md`](personality-system.md) **性格**、[`combat-effects-system.md`](combat-effects-system.md) **Trait（战斗天赋）** 并列，互不替代。

**设计目标**：同一职业、同一等级，两个角色的面板仍应有差异——差异来自 **属性天赋** + 开局选定的 **成长模式**（§4.5.3）。

#### 4.5.0 招募与培养哲学（无「废卡」）

> **记录日期**：2026-06-11 · 与 XCOM/BB 招募随机性一致，但刻意避免「抽到就删」的绝望感。

**原则**：招募 roll 可以差，但不应出现 **没有任何亮点** 的纯废人。只要有一项突出，配合培养路径（含随机模式下的 **凹点**），都应能长成可用战力。

| 亮点维度 | 例子 | 补偿路径 |
|---|---|---|
| **性格** | 忠诚 / 冷静 / 好色（事件选项优） | 营地事件、开战士气、结算钩子 |
| **属性天赋** | ★★ 慧心、双天赋 | 成长倾向 / 固定表定向加量 |
| **初始加值** | 招募面板某项特别高 | 即战力或 JP 省点 |
| **凹点（随机模式）** | 成长 roll 不佳 | 读档重 roll，属培养玩法 |
| **JP 技能** | 低 Melee 学 **借势合击**（§5.4） | 靠编队围攻补命中，见下 |

**不做的事**

- ❌ 隐藏「垃圾种子」或不可救的 stat 上限（体质天花板若存在，应仍留 JP/技能出口）
- ❌ 用单维 Melee 一锤定音「这人废了」

**与 §一红线**：「不能存在打不动人的上场单位」= 低面板角色必须有 **编队协同 / JP** 抢救空间，而非要求人人高 Melee。

#### 4.5.1 性别（Gender）

| 字段 | 说明 |
|---|---|
| `gender` | `male` / `female`（Phase 3 仅二值；立绘 / 语音 / 事件文案扩展预留） |
| 作用 | 影响 **属性天赋池抽样权重** + 少量 **初始属性微调**；**不锁职业**、不单独决定战斗 AI |

**人口层倾向**（可被个体例外覆盖）：

| 性别 | 天赋抽样偏重 | 初始微调（在职业基线之上） |
|---|---|---|
| **女** | Init、Wisdom 类属性天赋 | Init +0～+3、Wisdom +0～+3（随机） |
| **男** | Stamina、HP 类属性天赋 | max_stamina +0～+5、HP +0～+5（随机） |

**例外**：招募生成时保留 **~15%**「反倾向」roll——例如男性仍可能抽到「慧心」「迅捷」，女性仍可能抽到「铁骨」「厚血」。例外角色在名册 UI 可标「异禀」类氛围标签（可选，非必须）。

#### 4.5.2 属性天赋（Stat Affinity）

每人招募时获得 **1～3 个**属性天赋（终身不变；传奇 NPC 可定稿表）。

| 天赋 id | 显示名 | 绑定属性 | 说明 |
|---|---|---|---|
| `affinity_swift` | 迅捷 | Init | 先制 / 闪击向 |
| `affinity_keen` | 慧心 | Wisdom | 侦察 / 暴击 / 夹击向 |
| `affinity_iron` | 铁骨 | HP | 耐打 / 前排向 |
| `affinity_endure` | 刚毅 | Stamina（max_fatigue） | 持久战 / 重甲向 |
| `affinity_steel` | 稳阵 | Defense | 盾墙 / 枪阵向（中性，男女同权） |
| `affinity_edge` | 锋刃 | Melee | 输出向（中性） |

- **数量**：`roll_count = 1 + (20% 出 2 个) + (5% 出 3 个)`，上限 3。

- **与 Trait 区分**：Trait = 战斗规则被动；属性天赋 **只影响成长**，不进 `CombatEffectContainer`。

#### 4.5.3 双成长模式（战役 / 新游戏选项）

开局或战役设置二选一；**整局不可中途切换**（避免凹点跨模式）。

| 模式 | id | 成长规则 | 凹点 | 定位 |
|---|---|---|---|---|
| **固定成长** | `growth_fixed` | 每级按 **职业固定表 + 天赋定向加量** 确定性提升；同角色同级永远相同 | ❌ **无效**（无升级 RNG） | 硬核、可规划、偏 FFT/Berwick |
| **随机成长** | `growth_random` | 按 **属性天赋加权** 随机加点（擅长项期望更高）；FE 风逐项掷骰 | ✅ **允许**读档重 roll | 培养感、偏 FE/BB |

**共同规则**

- ❌ 均不开放 **手点**分配。
- ✅ 升级时均播放 **立绘 + 属性增量** 演出（固定模式数字确定但仍可展示「+2 Melee」动画）。
- 性别 / 背景仍影响初始面板与（随机模式下）成长率微偏（§4.5.1）。

**固定成长 — 规则草案**

```
每级增量 = class_level_table[job_id][level]     # 职业表，全员同级同职业相同基线
           + sum(affinity_flat_bonus[affinity]) # 天赋绑定属性 +N（★等级待定）
```

- 无掷骰 → 战前存档对 **升级结果** 无影响（凹点仅能对 **战斗过程** 有用，不能改成长）。
- 适合 speedrun / 硬核玩家 / 减少 RNG 焦虑（对照 [`references/berwick-saga.md`](references/berwick-saga.md)）。

**随机成长 — 规则草案**

- **倾向性随机**：每级有 budget 或逐项成长率；带「锋刃」者 Melee 权重 / 成长率显著高于无天赋者。
- 属性天赋（及待定 ★ 等级）→ 绑定属性 **+Δ% 成长率** 或 **加权 +Δ 点**。
- 升级演出：立绘 + 逐项 **+1 闪烁 / —**；间接属性 +1 时副文案换算（暴击/士气/气力，见 §4.5.4）。
- 读档重打 → 升级 RNG 重掷；**不禁止、不暗改**。

**待决（随机模式专属）**

| 议题 | 选项 | 倾向 |
|---|---|---|
| 属性多、战斗体感 | 每级至少 1 项战斗显性属性 +1 保底？ | 待原型 |
| 掷骰形态 | FE 逐项 % vs 每级 weighted 3 点 | 倾向 FE 逐项 |
| 成长率可见性 | 名册隐藏 vs FE14 式公开 | 随机模式可公开 |

#### 4.5.4 升级演出（两模式共用）

```
EXP 达标 → LevelUpScene
  → 立绘（Paper Doll / 半身）
  → 属性列表展示本次增量（固定=确定值；随机=逐项 roll 动画，可跳过）
  → 确认 → 写回 Stats
```

- **随机模式**：动画播完前不写盘，便于玩家决定是否读档（流程层不拦凹点）。
- **固定模式**：增量只读表，无 reroll 动机；UI 可简化动画。

## 五、职业技能体系

### 5.1 四层架构

```
武器层（物理工具）：
  damage_base / AP / range / block / weight
  → 武器不决定能用什么招，只决定物理可能性

通用攻击模式层（任何人都会，受武器物理限制）：
  普攻 / 瞄头 / 精准射击 / 全力一击
  → 不需要 JP 投入

武器专精层（JP 投资，绑定武器 tag）：
  横刀专精 / 短矛专精 / 长矛专精……
  → 进度 **永久保留**；转职时 **已学且新职武器池内有** → 保留激活，否则卸载（§5.3.2）

职业技能层（按 job_id 分轨 JP）：
  盾击、突击冲锋、架枪、袭扰……
  → **已学不丢**；跨职使用靠 **旧艺负载**（§5.3.2），深度 = 多职业 JP 分摊 + Wisdom 带宽组合
```

### 5.2 武器池与「手拙」规则

**规则**：
- 职业武器池定义了"能熟练使用的武器"（见 § 4.2）
- **池内武器**：无惩罚，可学习该武器专属技能（JP 投入后解锁）
- **池外武器**：强行装备触发「手拙」debuff，且永远无法学习该武器专属技能

**「手拙」debuff 效果**：
| 惩罚项 | 数值 |
|---|---|
| 气力消耗 | +10% |
| 命中率 | -10% |

> 设计意图：允许职业跨池装备（有趣的 build 选择），但要付出实际代价。重甲职业用匕首、斥候扛陌刀都是玩家的自由，只是需要接受手拙惩罚。

**JP 专精系统（武器池内）**：

| JP 投入 | 效果 | 典型 JP 成本 |
|---|---|---|
| 未投入 | 池内武器基线，无惩罚 | 0 |
| **初级专精** | 该武器气力消耗 ×0.8 | 100 |
| **高级专精** | AP -1、暴击 +5 | 200（需初级前置）|

> 池外武器无法投入 JP，即使到手拙状态也无法通过专精弥补惩罚。

### 5.3 技能互斥规则

- **一次攻击仅 1 个主动技能**：通用攻击模式 OR 职业技能，不能叠加
- **技能分支互斥**：标注"二选一"的技能升级路径不可兼得
- **被动效果可叠加**：职业被动 + 词条被动，但受反割草红线 #4 约束（总和 ≤ +30%）

#### 5.3.1 高级阵线技（镇阵）

> **模板**：阵列型职业在技能树 **~350 JP** 档位的 **主动** 锚点，与 **不屈**（被动、残血触发）互补——**镇阵** = 主动开「钉在原地、减伤、不被推走」。

| 字段 | 值 |
|---|---|
| 技能 | **镇阵** |
| 类型 | 主动（SELF） |
| AP | 2 |
| JP | 350 |
| 持续 | **1 回合**（至使用者 `end_turn`） |
| 适用职业 | 跳荡、枪兵、押衙亲兵等 **阵列/盾墙** 向（各职业树各挂 1 节点，效果相同） |

**效果**

1. 本回合承受 **HP 伤害 ×0.5**（甲伤分配后扣 HP 部分；经 `CombatEffect` `damage_taken_mult = 0.5`）。
2. **不可移动**：`move_ap` 视为 0 / 拒绝 `move_along_path`（仍可用不位移的主动技、攻击、格挡类）。
3. **免疫位移**：击退、拉拽、冲锋推挤、扫杆逼退等 **不改变本格**（与 `status-effects` 位移 tag 对接）。
4. **可攻击**：不锁普攻与近战主动（与 **吐纳** 整回合不同）。

**与相近技能**

| 技能 | 差异 |
|---|---|
| **不屈**（跳荡被动） | 残血自动 +防/士气；不禁位移 |
| **铁壁**（枪兵被动） | 未移动 +5 防；无减伤、无禁位移 |
| **盾牌格挡**（§5.12） | 仅 +block；可移动 |
| **镇阵** | 主动 −50% HP 伤 + 钉死格子 |

**实现落点**：Phase 2.5 `Ability` + debuff `rooted` + `displacement_immune` + fold `damage_taken_mult`；AI Defend 行为在 ZoC 缺口时提高权重。

#### 5.3.2 转职与技能保留（FFT Build 模型 v0.3）

> **记录日期**：2026-06-11 · v0.1「同级清空职业技能」**已废弃**；v0.2「固定跨职槽」**由 v0.3「旧艺负载」取代**。  
> **目标**：多职业 JP 投资 → **已学不丢** → **Wisdom 带宽限制兼修** → 杂交 build。

##### 核心原则（学 FFT，不学 FFT 成长 stack）

| 原则 | 说明 |
|---|---|
| **已学永久** | 在某 `job_id` 上花的 JP、解锁的节点 **不因转职删除**；死亡才归零 |
| **JP 分职业记账** | `job_jp[job_id]` 独立累计（`economy-system.md` §2.1） |
| **深度在组合** | 限制来自 **旧艺负载 / 互斥 / 武器池**，不是 wipe 技能 |
| **当前职业 = 身份** | 武器池、默认 chip 栏、**本职技能零负载** |

**不照搬 FFT**：不在多职业间 stack 基础属性成长；**只搬 JP 多轨 + 跨职兼修**。

##### 「本职记忆」与「旧艺负载」

> 防止 Wisdom 成为全员死绑属性：**本职不带带宽成本**，兼修才吃 Wisdom。

| 类别 | 定义 | Weight | 激活条件 |
|---|---|---|---|
| **本职记忆** | 当前 `current_job_id` 上 **已学** 的全部主动 / 被动 | **0** | 转职后：已学 **且** 属新本职 → 自动激活 |
| **武器专精** | 绑定 `weapon_tag` 的 JP 专精节点（视同技能） | **0** | 转职后：已学 **且** 新职 **武器池含该 tag** → 保留激活；否则卸载 |
| **旧艺（兼修）** | 在 **其他** `job_id` 学过、且挂载到当前 build 的技能 | **> 0** | `Σ 已激活兼修.weight ≤ unit.wisdom` |
| **通用池** | §5.4 等全职业共享节点 | **0** | 学过即 **常驻生效**；转职 **不卸载** |

**逻辑**：

```
本职 / 武器专精（池内）= 当前身份可用的肌肉记忆 → Weight 0 → 已学且新职匹配 → 自动激活
兼修 = 跨职挂载 → Weight > 0 → 转职时默认全部卸载，须手动重勾
通用 = 全职业共享 → 学过即生效，不进 inherited_skills，转职不卸载
武器专精（池外）= 进度在银行，激活卸载；换到含该 tag 的职业后自动恢复
```

**低 Wisdom 保护**：走卒 / 猛将 Wisdom 30 时，**本职树仍可全开**；兼修最多 ~2 个 T1 或 1 个 T2，**带不动 T3/T4**。

##### 技能重量（Weight）分级

> 基准区间 **Wisdom 30–120**；每个 `Ability` / 被动节点配置 `@export var weight: int`（本职节点填 **0**）。

| 档次 | 典型代表 | 建议 Weight | W30 能兼修 | W120 能兼修 |
|---|---|---|---|---|
| **T1 轻量** | 基础被动（+5% 命中）、小位移、简单包扎 | **10–15** | ~2 个 | ~8–12 个 |
| **T2 中量** | 核心战术主动（碎甲、横扫）、强力功能被动（不屈、铁骨） | **20–30** | ~1 个 | ~4–6 个 |
| **T3 重量** | 跨兵种大招、质变被动（单次免死、AP 消耗 −1） | **40–50** | ❌ | ~2–3 个 |
| **T4 终极** | 稀有资源解锁的终极奇术 | **60+** | ❌ | ~1–2 个 |

**配置原则**：

- 同一技能 **在本职 Weight=0**；从 **源职以外** 挂载时用表中档位（例：枪兵 **架枪** 在枪兵=0，铁骑兼修=25 T2）。
- 被动与主动 **同等计重**；`mutex_group` 姿态类兼修仍占带宽。
- 数值待 K3 逐条填表 + sim；上表为 **设计锚点**，可 ±5 微调。

##### 转职时发生什么

**同级互转**与**纵向晋升** **同一规则**。

**学习进度（银行）**：`job_jp[*]`、`learned_nodes[*]`、通用池 / 武器专精进度 **全部保留**，死亡才归零。

**战斗激活（上场）**：转职瞬间按下列规则 **重置激活集**：

| 技能来源 | 转职后激活状态 |
|---|---|
| **新本职** + 已学过 | ✅ **自动激活**（本职记忆，Weight 0） |
| **武器专精** + 已学过 + 新职 **武器池含该 tag** | ✅ **保留激活**（Weight 0） |
| **武器专精** + 已学过 + 新职 **武器池不含该 tag** | ❌ **卸载**（进度保留，换职后可恢复） |
| **旧本职 / 其他 job** + 已学过 | ❌ **默认全部卸载** → 旧艺库，须手动重勾（占带宽） |
| **通用池**（§5.4 等）+ 已学过 | ✅ **保持激活**，不卸载 |

```
switch_job(new_job_id):
  learned[*]                              // 银行不动
  inherited_skills = []                   // 跨职旧艺：默认清空
  auto_activate(current_job learned)      // 本职：已学且属新职 → 激活
  for tag in learned_weapon_mastery:
      if tag in new_job.weapon_pool: activate(tag)
      else: deactivate(tag)               // 池外专精：卸载
  // learned_universal：维持激活，不卸载
```

**UX**：

- 不弹「失去 N 技能」；可轻提示「旧艺与池外专精已卸载，可在兵籍重新装配」。
- **旧艺空载提醒**：当 `inherited_skills` 为空（已用带宽 = 0）**且** 旧艺库存在 ≥1 项 **当前可装配** 候选时，兵籍「旧艺负载」栏显示 **红色感叹号 `!`**；hover「尚有旧艺未装配」；装配任意旧艺或无可用候选时消隐。
- 通用池 **不出现** 在卸载列表里。

**纵向「全覆盖」** = 上位树含更强版节点，**不是** 清空下级银行；下级节点卸载后仍在旧艺库，可兼修带回。

**示例**

```
转职前：铁骑，兼修 [枪兵·架枪]；已学 骑枪专精、横刀专精；通用 [借势合击]

转职 → 陌刀手（池：陌刀、横刀，无骑枪）：
  保留激活：陌刀本职已学 + 横刀专精 + 借势合击
  自动卸载：架枪 → 旧艺库；骑枪专精 → 进度在、激活关闭
  UI：旧艺库有架枪等可选项 → 旧艺栏显示红色 !
```

> **设计立场**：允许 FFT 式 hybrid，但 **兼修带宽** + **JP 稀缺** 自然封顶；超模来自 **4 人分工**，不是 1 人全能。

#### 5.4.1 借势合击（合击眼力 — 低 Melee 命中抢救）

> **Phase 2.5 / Phase 3** 实装；与全局 **Overwhelm（围攻）** 叠加，SoT：`weapon-system.md` / `DamageSystem`（每相邻友军 +5% 命中，全局 cap +20%～+25%）。

**设计意图**（§4.5.0）：Melee 偏低的新兵 / 招募 roll 不应「永远打不中」——愿意 **配队集火** 的人，用 JP 买命中出口。

| 字段 | 值 |
|---|---|
| 类型 | 被动（通用 JP 池，任意战斗职业可学） |
| JP | 200 |
| 互斥 | 与跳荡 **野战八方**（受击侧无视围攻，见 §5.5 跳荡 **野战八方** 说明）无冲突——本技能只改 **你打别人** 时的围攻加值 |

**规则（草案）**

1. 仅当 `overwhelm_count >= 1`（至少 1 名友军与你在同一目标上形成围攻）时生效。
2. **个人围攻倍率**：你的围攻加值按 **+8% / 参与友军** 计算（替代基础的 +5%/人，只改 **持有者** 这一击，不改队友）。
3. **个人 cap**：持有者围攻部分最高 **+24%**（3 友军即满）。
4. **示例**：Melee 55 打 80 防目标，裸命中 ~5% 下限；2 友军围攻 → 个人围攻 +16% → 约 21%，仍低但可玩；再叠高地 / 状态 / 第三友军至 +24%。

**AI / UI**

- AI：低 Melee + 已学本技能 → Engage 评分提高「等友军贴脸再出手」权重。
- UnitTooltip：持有者学技能且 `overwhelm_count >= 1` 时，预览行显示 `[合击+XX%]`（仅围攻部分）。

**实现落点**：`DamageSystem.calculate_hit_chance()` 读 `attacker.has_passive("jie_shi_he_ji")` → 仅调整持有者 `overwhelm_bonus`（+8%/人、cap +24%）；Phase 2.5 经 `PassiveEffect` / J5 注册。

#### 5.4.3 身法精通（位移技能 AP / 气力减免）

> **记录日期**：2026-06-13 · brainstorm 池；**Phase 2.5+** 与 `PassiveEffect` / 位移 `Ability` 一并实装。  
> **与身轻如燕区分**：身轻如燕改 **闪避**（`dodge_pts`）；本技能改 **位移类主动/utility 的 AP 与气力**，不叠 dodge。

| 字段 | 值 |
|---|---|
| 暂定名 | **身法精通** |
| 类型 | 被动（通用 JP 池，任意战斗职业可学） |
| JP | 250（占位） |
| 状态 | **备忘** — 未排期 |

**效果（草案）**

1. **AP**：对标记为 `displacement` 的技能，结算 AP 消耗 **-1**（**下限 1**；原 2→1，原 1 不变）。
2. **气力**：同上技能，发动时附加气力消耗 **×0.8**（叠乘职业/武器专精的气力减免，不叠乘两次「身法」类被动）。
3. **覆盖范围**（`Ability.tags` 含 `displacement` 或等价枚举）：
   - 通用：**滑步**、**换位**
   - 位移池：**擒拿拉拽** / **推撞** / **牵制** / **战术跳跃** / **冲锋推退** / **绊摔** 等 §六 位移类
   - **不含**：纯 hex 移动（`AP_PER_HEX` 走位）、**吐纳**、**先发制人**、无位移效果的 debuff 技

**设计意图**

- 轻甲 / 跳荡 / 不良人 build：用 JP 换 **更便宜的控场与脱战**，与身轻如燕（闪避 T）形成「身法双投」但不强制绑定。
- 与 **滑步**（2 AP）联动：学身后 1 AP 滑步，仍可在同回合攻击（若剩余 AP 够），强化「打了就跑」。

**互斥 / 约束**

- 与 **身轻如燕** **可并存**（一 dodge、一位移成本，职责不同）。
- 反割草：仅减 **位移类** 成本，不改普攻/重击伤害；AP -1 下限 1，避免 0 AP 无限位移。

**实现落点**：`Ability.get_ap_cost(user)` / `Ability.get_fatigue_cost(user)` 读 `user.has_passive("shen_fa_mastery")`；位移技能注册时打 `displacement` tag。AI Engage/Disengage 估值同步扣 AP 机会成本。

#### 5.4.5 包扎（初级 / 高级）与巧手

> **Phase 2.5 / Phase 3** 实装；Debuff / 临时伤势 SoT：`status-effects.md`。  
> **巧手**：与 CombatMenu **I** 置物架 / 武器切换流程对接（0 AP，仍占「切换动作」一次）。

##### 巧手

| 字段 | 值 |
|---|---|
| AP | **0**（切换本身不耗 AP） |
| JP | 60 |
| 效果 | 在 **背包已装** 的主武器间切换（二选一或多把，以 `Unit.inventory` 为准） |
| 限制 | 建议 **每回合最多 2 次**；切换不触发借机；切换后本回合仍可攻击（若 AP 够） |

##### 包扎·初级

| 字段 | 值 |
|---|---|
| AP | 2 |
| JP | 80 |
| 目标 | 自身或 **相邻 1 格** 友军 |
| 池子 | 目标身上存在的：`流血` / `毒性（轻|重）` / `燃烧` / `无力`（`status-effects.md` §二） |

**流程（草案）**

1. 从目标 **当前持有的上述 debuff** 中 **随机抽 1 项** 作为本次尝试对象（无则失败，仍耗 AP，日志「无可解之症」）。
2. 解除成功率：`P_cure = clamp(35% + (施术者.Wisdom − 40) × 1.0%, 20%, 95%)`。
3. 成功 → 移除该 debuff 实例；失败 → 保留，日志 `[包扎未效]`。
4. **不含**：眩晕 / 缴械 / 士气档 / **临时伤势本体**（伤势仍挂在单位上，仅高级可暂蔽其效果）。

**Wisdom 示例**

| Wisdom | 解 DOT/无力 成功率 |
|---|---|
| 40 | 35% |
| 60 | 55% |
| 80 | 75% |
| 100 | 95% |

##### 包扎·高级

| 字段 | 值 |
|---|---|
| AP | 3 |
| JP | 180（需 **包扎·初级**） |
| 目标 | 同初级 |
| 额外 | 对目标施加 buff **`伤势暂蔽`**（整场战斗，见 `status-effects.md` §1.1） |

**暂蔽个数（随机 + Wisdom 偏置）**

```
K_base = 1 + floor(Wisdom / 35)          // Wis 70→2, Wis 100→3
K = clamp(K_base + randi_range(-1, +1), 1, 目标临时伤势总数)
```

- 从目标 **临时伤势列表**（§四）中 **随机选 K 条**，其 **战斗内 debuff 效果** 在本场战斗中 **不生效**（伤势条目仍在，脱战/城镇治疗逻辑不变）。
- 已暂蔽的伤势 **不可重复选**；同目标 **伤势暂蔽** buff 不叠加，后施覆盖前次（刷新 K 与列表）。
- **不含**：永久伤势；不解除 debuff 池中的流血/毒/火（仍靠初级 roll）。

**实现落点**：`BandageBasicAbility` / `BandageAdvancedAbility`；`CombatEffect` `injury_effects_suppressed: Array[injury_id]` + fold 时跳过对应修正。

#### 5.4.2 设想技能（不定稿 · 未必实装）

> **记录日期**：2026-06-11 ·  brainstorm 池；未排期，平衡与 `design.md` §三「气力无被动恢复」原则冲突时需再审。

##### 承锋纳气（受击回气）

| 字段 | 值 |
|---|---|
| 暂定名 | **承锋纳气**（备选：**砺战养气**、**化劲回元**） |
| 类型 | 被动（通用 JP 池设想） |
| JP | 320（占位） |
| 状态 | **设想** — 不承诺实装 |

**效果（草案）**

- 触发：每次 **被近战攻击检定**（命中 / 未中 / 格挡闪避均算「承锋」一次，与 `DEFEND_FATIGUE_BASE` 同触发点）。
- 原规则：防守方 `add_fatigue(calculate_defend_fatigue)`（基础 2 × 重量系数）。
- 本技能：**不扣** 上述防守气力，改为 **`+4` 气力**（从 `fatigue` 减少，即剩余气力上升；不超过 `max_stamina`）。
- **每回合上限**：靠本技能最多回 **+12** 气力（3 次受击触顶）。
- **互斥**：与 **气海回旋** 二选一（气海回旋是「多扣气换减伤」，本技能是「挨打回气」）。

**设计意图**

- 前排 / 盾墙 **故意吃刀** 的 build：敌人集火你反而帮你维持气力档，与「集火加速力竭」形成反制窗口。
- 风险：鼓励卖血站桩；需配合 **不能回 HP** 红线，且 AI 可能学会绕开。

**与全局原则的张力**

- `design.md` §三：气力 **无被动自然恢复** — 本技能是 **条件触发型**，类似「被打才回」，若实装需写入例外白名单。
- 若试玩 feel 过强：可改为「仅 **未命中** 时回气」或「仅 **格挡/闪避成功** 时回气」，削弱卖血最优。

**实现落点（若采纳）**：`Unit` / `DamageSystem` 防守气力分支 → `PassiveEffect.on_defend_checked()`；战斗日志 `[承锋+4 气力]`。

> 铁骨与气海回旋互斥（总甲重不同时自动生效）。

> ⭐ = Phase 2 已实装（`Unit.use_ability_*` + CombatMenu chip）；其余为设计占位。

#### 5.10.1 长虹贯日（直线穿透刺击）

> **定位**：贴脸一枪穿两人——用**阵型纵深**换伤害，与长矛普攻 2 格、架枪反骑、挑盔点头形成三角差异。  
> **互斥**：`mutex_group = "attack"`，与普攻 / 挑盔刺 / 破阵突刺等同回合二选一。

**基础参数**

| 字段 | 值 |
|---|---|
| `ability_id` | `changhong_guanri` |
| Kind | `ATTACK` |
| Targeting | `LINE_PIERCE_ENEMY`（见 `ability-system.md` §2.2） |
| 武器 | 长矛 `spear`（双手） |
| 攻击模式 | 强制 `pierce` |
| 射程 | **仅 1**（覆盖武器 `range: [1,2]`，本技能不可隔格点选） |
| AP | 武器 `ap_cost` + 2（长矛默认 6 → **8**） |
| 隐藏气力 | +8（与挑盔刺同级） |
| JP | 240 |

**目标解析（六边形直线）**

1. 玩家 / AI 点选**主目标** `T`，须满足 `distance(attacker, T) == 1`。
2. 穿刺方向 `dir = T - attacker`（相邻格方向唯一）。
3. **次目标格** `S = T + dir`（从攻击者经主目标向后的第 1 格）。
4. 若 `S` 上有**存活敌军**单位 `U₂`，则 `U₂` 为次目标；否则仅打主目标。
5. **不穿透友军、不穿透障碍**；次目标格无单位则无第二段伤害。
6. **仅穿透 1 格**：`S` 之后再远的单位不受波及。

示意（`A` 攻击者，`E1` 主目标，`E2` 次目标，`·` 空）：

```
      ·   E2   ·
         E1
    ·    A    ·
```

**伤害与命中**

| 目标 | 伤害系数 | 命中 |
|---|---|---|
| 主目标 `T` | ×1.0（正常长矛 pierce 管线） | 正常 melee 命中公式 |
| 次目标 `U₂` | **×0.75**（`hp_mult`） | 独立命中检定；额外 **-5%** 命中（远段枪尖难控） |

- 两段各自掷头部部位、暴击、护甲渗透；**主目标未命中不取消次目标检定**（枪尖已送出）。
- 次目标若在主目标**背后**（相对攻击者），不额外享受背刺被动——本技能自带穿透，避免叠乘过强。
- 借机攻击（OA）、ZoC 规则与普攻一致；技能不附带位移。

**UI / 反馈**

- 高亮：主目标邻格 + 若存在合法次目标则高亮 `S` 格（淡色穿透预览线 `A→T→S`）。
- 战斗日志：`长虹贯日 → [主名]（命中/未中）`；有次目标时追加 `→ [次名]（75% / 命中/未中）`。
- 音效：一次突刺 + 第二段命中时短促二次音（可与 `combat_audio` 穿透层复用）。

**AI 估值（EngageScorer 扩展）**

```
score += EV(primary) + 0.75 * EV(secondary)   # 仅当 S 格有敌军时
```

优先：主目标低血可击杀，或次目标为弓手 / 医官等后排。

**实现要点（Phase 2.5）**

- `HexCoord.extend_along(attacker, through, steps=1) -> Vector2i` 返回 `S`。
- `Ability.resolve_targets()`：`LINE_PIERCE_ENEMY` 返回 `[T]` 或 `[T, U₂]`。
- `ChanghongGuanri.apply()`：循环 `DamageSystem.execute_attack`，次目标 `options.hp_mult = 0.75`、`options.hit_modifier -= 5`。
- `can_target(hex)`：`distance == 1` 且 hex 上有敌军。

**克制与风险**

- **强**：敌方纵队、两人叠在一条六向直线上（常见「前排盾 + 后排弓」）。
- **弱**：敌人扇形散开、侧翼包夹时无次目标；AP 8 高于普攻，空戳惩罚大。
- **与架枪**：架枪守位、长虹贯日主动破阵——同一职业不同回合角色。

| **牵制** | 标记目标：下回合目标首次移动消耗 +2 AP | 2 | 任意 | 不消耗移动，纯 debuff |
| **战术跳跃** | 跳跃到 2 格范围内空位（无视 ZoC/地形）| 4 | — | 落点需可见+无单位；轻甲（weight ≤ 12）限定 |
| **冲锋推退** | 移动 ≥2 格后攻击命中，推退目标 1 格 | 5 | 长矛/战斧 | 仅限冲锋攻击模式触发 |
| **绊摔** | 近战攻击附加：命中后 40% 概率目标下回合 Move -2 | 3 | 长棍/长矛 | 被推目标若 Move=0 则倒地 1 回合 |

**与地形交互的联动**（详见 [design/terrain-system.md](terrain-system.md)）：
- 推入水泽 → 减速 + 困住（每回合需通过检定才能移动出）
- 推入悬崖/高地边缘 → 坠落伤害
- 推入火墙 → 即时火焰伤害
- 推入拒马 → 伤害 + 停留
- 擒拿拉出危险地形 → 救援

**反割草约束**：位移技能本身不造成直接伤害（伤害来自地形交互），遵守"环境击杀 ≠ 技能伤害"原则。推撞/拉拽的命中检定参与减法制，不是必中。

## 七、人物背景（不是职业，是出身）

民族 / 区域 / 出身作为**人物背景**而非职业，与战斗职业完全解耦。每个角色随机生成时获得 1 个背景，影响起始属性、外观、**可触发的营地随机事件池**（见 `camp-events-system.md` §二），但**职业可以自由选择**。

> **背景 ≠ 性格**：背景决定「会遇到什么事」；性格决定「事件里有哪些回答选项」；选项影响隐藏 **好感度** → 开战士气 / 特殊事件 / 结局。

| 背景 | 起始属性偏向 | 装备/外观 | 事件 / 剧情 |
|---|---|---|---|
| **沙陀血统** | Init/Melee +5 | 沙陀刀、辫发 | 沙陀骁将线；`evt_shatuo_*` 事件池 |
| **党项血统** | Init/Ranged +5（弓加成） | 复合弓、皮裘 | 拓跋家族线；党项故旧事件 |
| **吐蕃血统** | HP/Defense +5 | 藏刀、皮盾 | 河湟线；吐蕃僧俗事件 |
| **粟特商人** | Wisdom/Resolve +5 | 西域弯刀、多语言 | 商路/互市/走私事件（高权重） |
| **回鹘血统** | Init/Ranged +8（顶级骑射） | 复合弓 | 回鹘牙帐线 |
| **汉人世家** | Resolve/Wisdom +5 | 文锦袍、家传剑 | 朝廷文书、门第荣辱事件 |
| **募兵出身** | HP +10（基线韧性） | — | 通用军营琐事（兜底池） |

## 八、后勤职业（6 个 NPC，不上场）

不参战 = 不受永久死亡约束 = 不需要平衡战斗数值。
**关键约束**：后勤占小队总名额，玩家必须在"6 战斗员 vs N 战斗员 + M 后勤"之间做战略权衡。

| 后勤职业 | 唐代名 | 核心功能 | 系统联动 |
|---|---|---|---|
| **军医** | 医户/药童 | 营地修养 +30~100% / 死亡判定改判 / 解永久创伤 | 受伤、永久死亡 |
| **方士** | 炼丹师 | 开启炼金系统（金疮药/毒药/火油/烟雾弹等）| 消耗品经济 |
| **铁匠** | 军器监工匠 | 武器/护甲修理 / 传说级锻造 / 耐久延长 | 装备经济 |
| **道士** | 钦天监道人 | 战场天气预测/改造 / 占卜吉凶 | 战场地形 |
| **军师** | 押衙参谋 | 行军速度 / 事件预警 / 敌方阵容情报 | 大地图战略 |
| **乐师** | 教坊乐工 | 营地 morale 恢复 / 招募成功率 / 特殊曲目 buff | 招募、士气 |

**雇佣机制**：
- 大地图城市/据点找到不同稀有度的后勤 NPC
- 每周支付工资（比战斗员低 50-70%）
- **占小队名额**：典型 12 人 = 6 战斗 + 6 后勤 / 或 8 战斗 + 4 后勤
- 多个同类后勤不叠加（取最强）→ 6 个名额最多 6 类各 1 个
- 可解雇 / 替换 / **等级培养**（学徒 → 熟练 → 大师，影响效率）
- 深度 **专属培养树**（花资源解锁能力 + 特色事件）见 `camp-meta-systems.md` §二（Phase 4，与等级线可叠加）

**关键原则**：
- ❌ 后勤 NPC 永远不能上场战斗
- ❌ 后勤 NPC 不上战场则不死（可能因事件离队）
- ✅ 多个同类不叠加 → 决策聚焦"养哪 6 类"
- ✅ 后勤等级影响生活系统效率，不影响战斗平衡

## 九、JP / EXP 系统（摘要，详见 [design/economy-system.md](economy-system.md)）

### 9.1 双轨制

| 资源 | 作用 | 来源 |
|---|---|---|
| **EXP** | 提升基础属性（HP/Init/Melee 等）| 击杀/参与战斗 |
| **JP** | 解锁职业技能/Trait | 击杀/参与战斗（按职业积累）|

### 9.2 稀缺性设计

- **EXP 池固定**：不能 grind，全游戏总 EXP ~12000-18000
- **JP 池固定**：单角色一生 JP ~3000
- **Mastered 成本**：单职业 ~1500 JP → 一辈子能 Mastered 2 个职业
- **死亡归零**：阵亡 → 所有 EXP/JP 永久归零

### 9.3 职业等级（JP → Level）

|---|---|---|
| Lv 1 | 0 | 基础武器权限 |
| Lv 2 | 100 | 初级技能 |
| Lv 3 | 250 | 中级技能 |
| Lv 4 | 450 | 高级技能 |
| Lv 5 | 700 | 被动专精 |
| Lv 6 | 1000 | 职业精通 |
| Mastered | 1500 | 全技能解锁 + 可作副职业 |

## 十、弩手专属机制（详解）

弩手不是 FFT 派的"脆皮远程"，是**会射重弩的重步兵**：

- 装备：擘张弩（主武器，远程）+ 横刀（副武器，近战自卫）+ 中重甲
- HP 100-120 / Init 70-90（低）/ Move 3-4（重甲扣）
- **上弦机制**：弩发射后进入"未上弦"状态，必须用 3 AP 上弦 → 节奏慢，单发伤害是弓的 2 倍
- **抵近射击**：1-2 格距离发射 → 伤害 +50%（贯穿铠甲）→ 被近身反而是机会
- **盾手协同**：旁边有盾兵/枪兵友军 → 弩手 eff_def +10
- **近战兜底**：被多敌围住 → 切横刀 + 格挡架势硬扛

### "弓手 build"如何实现（无专职弓手）

- 选基础职业（奇兵 / 斥候 / 游侠 / 跳荡 等）
- 装备步弓为主武器
- 投资 **bow_mastery 通用 JP 被动**
- 高 init + 高 melee + 满 bow_mastery → 准头堪比专业
- 战术体验：节奏快（无上弦）、伤害低（25-40）、机动性强 → 与弩手"节奏慢、伤害高、阵列站定"完全不同

## 十一、防御体系与职业定位

### 11.1 合一检定公式

> **SoT**：`weapon-system.md` §6.1.1（v3.2，2026-06-12）。本节旧 `effective_defense` 描述已废弃。

```
Hit% = clamp((atk_skill - final_def) / 100 + hit_bonus, 5%, 95%)

final_def = apply_high_def_penalty(composed × def_mult)
composed  = base_def + dodge_pts + block_pts + def_flat
dodge_pts = 默认 0；身轻如燕 max(0,init−全身重)×20%；技能 +N flat
block_pts = 配盾仅盾 / 双持主+副 block 之和 / 否则主手（与 defense 无关）
def_flat  = 气力档 −5/−10 等 flat 修正
def_mult  = Π(1 + defense_pct)（士气档等；H2 前 stub ×1.0）
hit_bonus = 围攻 + 高地 + 攻方 buff + 技能 + 天气占位
```

Miss 归因：同一次 `roll` 在 miss 带内按 `防御 : 闪避 : 格挡`（`base_def+def_flat` / `dodge_pts` / `block_pts`）切分；某池为 0 则不出现对应归因（见 §6.1.1 C）。

### 11.2 远程修正（待实装）

```
if 攻击者是远程:
    block_pts 不变              // 盾挡箭
    dodge_pts × 0.5（占位）     // 箭难闪；见 weapon-system §6.1.1 options
```

### 11.3 防御三角（v3.2 示例，含身轻如燕）

| 定位 | 典型配置 | block_pts | dodge_pts | 备注 |
|---|---|---|---|---|
| **盾T** | 札甲+团牌+横刀 | **25**（仅盾） | 0 | 武器 block 不计 |
| **闪避T** | 皮甲+双匕，init100 wt18，已点身轻如燕 | 6（3+3） | **16.4** | 未解锁则 dodge=0 |
| **重甲无盾** | 札甲+双手战锤 | 0 | 0 | 靠护甲池硬抗 |
| **均衡型** | 锁甲+横刀，无 JP 闪避被动 | 8 | 0 | defense 只在 base_def |

## 十二、「本职记忆与旧艺负载」—— 实现附录

> **设计 SoT**：§5.3.2 v0.3。本节仅列 **代码 / 数据落点**。

### 12.1 核心规则（摘要）

| 规则 | 说明 |
|---|---|
| **本职记忆 Weight = 0** | 当前 `job_id` 已学技能不占 Wisdom |
| **旧艺 Weight > 0** | 其他 `job_id` 学来且 **激活** 的技能计入负载 |
| **总负载 ≤ Wisdom** | `Σ active_inherited.weight ≤ unit.wisdom` |
| **武器专精 Weight = 0** | 新职武器池内且已学 → 激活；池外 → 卸载（进度保留） |
| **转职默认卸载旧艺** | `switch_job` 清空 `inherited_skills`；按规则重算本职 / 武器 / 通用 |
| **旧艺空载 UI** | 带宽 0 且有可装配旧艺 → 红色 `!` |

### 12.2 心智负载分级

见 §5.3.2「技能重量分级」表（T1–T4）。

### 12.3 落到代码

```gdscript
# Ability.gd —— 新增字段
@export var weight: int = 0  ## 兼修负载；本职使用时视为 0，跨职激活时用配置值

# Unit.gd —— 新增字段 + 方法

var active_weapon_mastery: PackedStringArray = [] ## 当前激活的武器专精 tag

func get_wisdom() -> int
func get_inherited_bandwidth_used() -> int
func get_bandwidth_remaining() -> int
func can_activate_inherited(id: String) -> bool
func activate_inherited_skill(id: String) -> bool
func deactivate_inherited_skill(id: String)
func needs_inherited_attention() -> bool:
    ## 带宽为 0 且旧艺库有可装配候选 → true（UI 红色 !）

func switch_job(new_job_id: String) -> void:
    current_job_id = new_job_id
    inherited_skills.clear()
    _auto_activate_current_job_learned()
```

### 12.6 与原有系统的边界

| 系统 | 关系 |
|---|---|
| **JP 系统** | JP = 能不能学；旧艺负载 = 转职后能不能 **激活** |
| **武器专精** | 视同技能；转职按 **新职武器池** 保留或卸载；**Weight 0**（池内） |
