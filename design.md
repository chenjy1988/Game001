## 目标 & 优化点

### 优化点
- **战斗**：高度拟真的中世纪战斗模拟，每个数值都可被玩家理解和反推
  - 可以变成卡通画风
  - 战斗地形可以更多变
  - 掩体可以更多（但是可以破坏）
  - 高地差可以造成伤害
  - 攻击可以被格挡和闪避，如果持有盾，格挡概率更高，如果没有盾，闪避率更高
- **数值**：硬核但不劝退，暴露完整公式给玩家
- **节奏**：回合制但战术深度高，单场战斗可持续 10-20 回合，节奏可以更快一点

### 系统功能想法
- 炼金系统
- 锻造系统
- 建设系统
- 野兽掉落，提升收益，特殊掉落物低，用来制作装备、药水和道具
- 商店系统
- 指挥官系统：不同指挥官可以使用不同战场能力，例如战场视野、战场控制、战场干扰、战场治疗等
- 提升战斗时治疗的效果，可以避免伤势的 debuff，但是取决于角色能力和天赋
- 俘虏系统：可以俘虏敌人，然后可以使用俘虏获取酬金和道具，士气和血量越低，俘虏越高
- 援军系统

## 已实施

### 战斗节奏

| 项 | 取值 | 说明 |
|---|---|---|
| 移动 AP/格 | 2 | 9 AP / 回合 → 最远走 4 格 |
| 移动疲劳/格 | 3 | |
| 头部命中率 | 25% | 战兄弟同款；伤害 ×1.5 |
| 命中率上下限 | 5% / 95% | clamp 区间 |
| 借机攻击触发条件 | **本格被敌方控制即触发**（每步都触发，包括 ZoC 内平移） | 比战兄弟原版更严格 —— 战兄弟仅"离开"才触发 |
| 借机攻击命中行为 | **命中即停 + 退还本步 AP/疲劳** | 让"在 ZoC 内移动"成为高风险决策 |
| 借机攻击触发者代价 | 完全免费 | 不消耗触发者的 AP/疲劳 |
| 远程武器是否产生 ZoC | 否 | 只有 `weapon_type == "melee"` 才产生 |

### 随机系统

- 单一 d100 滚点（`randf()` 后比 hit_chance）
- **暴击**：命中 roll < `5% + 武器 bonus_crit_chance` 视为暴击，HP 伤害 ×武器 crit_mult
    - 倍率：匕首 1.8 / 战斧 1.6 / 剑/矛 1.5 / 战锤 1.3
    - 额外暴击率：匕首 +10%（匕首总暴击阈值 = 15%），其他武器 +0%
- 头身判定独立 d100 (25% / 75%)
- 伤害 `randi_range(min, max)` 均匀分布
- **战斗日志公开 chance 与 roll**（战兄弟风格："几率:60，掷出:42"）

### 视觉风格（B 方案：保持 2D 但升级精致度）

- 六边形：双层叠色模拟内阴影 + 顶部高光、暗金边框
- 高亮：移动/攻击 alpha 呼吸动画；借机攻击格 双层描边 + 急促呼吸
- 路径：金色圆点 + 末端更亮，随呼吸缩放
- 单位：阴影从单层换成柔光+实芯双层；底色暗+顶部高光圆+白光点 三层叠出立体感
- 当前回合单位：阵营色外圈柔光 + 头顶金色 ▼ 箭头 (呼吸上下浮)
- HP 微条：黑底 + 红/橙/深红三色随 HP 比例变
- 战场背景：Vignette shader 暗角（中心亮、四角暗）
- 攻击反馈：攻击者向目标方向 lunge → 回位；目标短暂左右抖动 + modulate 闪红（miss 闪灰）；暴击/重击触发屏幕震动

### AI

- 评分式决策，每回合枚举 (可达格 × 可攻击目标) 全候选
- 评分项：
  - 命中率 × 期望伤害 × 可攻击次数
  - 击杀奖励 +50
  - 低 HP 集火权重 (+50% per HP missing)
  - 借机攻击惩罚 −35/次（命中即停规则下加重）
  - 落点处于额外 ZoC 惩罚 −6/重叠
  - 移动距离微惩罚 −0.4/格

## 待讨论 / 待实现

### 🔵 Phase 2 内还要做的

#### 1. 围攻 Overwhelm（压制）
战兄弟规则：每个相邻友军给攻击者 +5 命中（被多人围殴时命中飙升）。
- 实现位置：`DamageSystem.calculate_hit_chance` 末尾加上 `+5 × 相邻友军数`
- AI 评分要考虑这条，会主动绕侧
- **需确认**：上限是否要 +20（4 个邻居就够），还是无上限？

#### 2. 伤害 ±25% 浮动
当前是 `randi_range(min, max)` 已经是均匀分布，**这条已经隐式做了**，只是浮动幅度由 weapons.json 中 `damage_min/max` 决定。
- **已自动满足**——无需额外做

#### 3. 士气 / 决心系统（六级状态）
战兄弟有：自信(Confident) / 稳定(Steady) / 动摇(Wavering) / 崩溃(Breaking) / 溃逃(Fleeing)
- 触发器：友军死亡、近距离致命一击、被压制时
- 数据已就绪：`Stats.resolve = 40`
- 影响：动摇时命中/防御 −10，崩溃时无法行动一回合，溃逃时强制远离
- 工程量较大，需要：UI 标签 + 决心检定流程 + AI 感知
- **需确认**：先做"决心检定 + 三级状态(自信/动摇/逃跑)"简化版，还是直接全六级？

#### 4. 高低差（Elevation）
- HexGrid 增加 `elevation: int` 数据
- 高位攻击者 +10 命中、远程射程 +1
- 高位被爆头率 +5
- 渲染上需要给不同 elevation 的格子分色
- **工程量最大**，看起来是 Phase 2 末尾或 Phase 2.5 才做的
